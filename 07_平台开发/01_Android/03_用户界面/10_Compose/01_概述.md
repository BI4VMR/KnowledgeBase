# 简介
Jetpack Compose 是一种用于构建用户界面的框架，基于响应式编程模型与 Kotlin 函数式语法，通过 Skia 图形引擎绘制图像，与传统的 View 体系相比，语法更为简洁、性能更高。

在 View 体系中，我们需要监听外部事件（例如：回调方法），并在事件发生时控制相关 UI 组件作出响应（例如：更新文本内容、控制图片透明度等）；在 Compose 体系中，我们通过函数描述 UI 组件，将可变属性（例如：文本内容、文本颜色等）作为函数的参数，每当外部状态变化时， Compose 运行环境将会自动调用受影响组件的声明函数，传入新的参数以重新绘制界面。

# 基本应用
## 旧版本
在较新版本（组件版本 `1.6.8` 、 BOM 版本 `2024.06.00` 及更高版本）中， Gradle 配置出现了变化，该部分将在后文章节 [🧭 新版本](#新版本) 中介绍。

🔴 示例一：构建 Jetpack Compose 环境。

在本示例中，我们为工程添加 Jetpack Compose 的相关配置，并创建基本的示例界面。

第一步，在模块的 Gradle 配置文件中添加对 Jetpack Compose 组件的依赖声明。

`build.gradle` :

```groovy
dependencies {
    // Jetpack Compose BOM 版本配置
    implementation platform('androidx.compose:compose-bom:2023.06.01')

    // Jetpack Compose 核心组件
    implementation 'androidx.compose.ui:ui'
    // 仅包含 `@Preview` 注解声明，需要在所有 Build Variant 引入确保编译通过。
    implementation 'androidx.compose.ui:ui-tooling-preview'
    // 2D 画布等基础 API ，被 `ui` 组件依赖，无需手动声明，仅编写基础库时需要。
    implementation 'androidx.compose.ui:ui-graphics'
    // 基础控件，被 `material3` 组件依赖，无需手动声明，仅编写基础库时需要。
    implementation 'androidx.compose.foundation:foundation'
    // Material2 控件
    implementation 'androidx.compose.material:material'
    // Material3 控件
    implementation 'androidx.compose.material3:material3'
    // Activity 支持
    implementation 'androidx.activity:activity-compose:1.6.1'

    // Jetpack Compose 调试工具
    debugImplementation 'androidx.compose.ui:ui-tooling'

    // Jetpack Compose 测试工具
    androidTestImplementation(platform('androidx.compose:compose-bom:2023.06.01'))
    androidTestImplementation 'androidx.compose.ui:ui-test-junit4'
    androidTestImplementation 'androidx.compose.ui:ui-test-manifest'
}
```

上述内容也可以使用 Kotlin 语言编写：

`build.gradle.kts` :

```kotlin
dependencies {

    implementation(platform("androidx.compose:compose-bom:2023.06.01"))

    // 仅包含 `@Preview` 注解声明，需要在所有 Build Variant 引入确保编译通过。

    // 2D 画布等基础 API ，被 `ui` 组件依赖，无需手动声明，仅编写基础库时需要。
    implementation("androidx.compose.ui:ui-graphics")
    // 基础控件，被 `material3` 组件依赖，无需手动声明，仅编写基础库时需要。
    implementation("androidx.compose.foundation:foundation")
    // Material2 控件
    implementation("androidx.compose.material:material")
    // Material3 控件

    // Activity 支持
    implementation("androidx.activity:activity-compose:1.6.1")

    // Jetpack Compose 调试工具
    // IDE 预览能力，通过 `debugImplementation` 引入即可。
    debugImplementation("androidx.compose.ui:ui-tooling")

    // Jetpack Compose 测试工具
    androidTestImplementation(platform("androidx.compose:compose-bom:2023.06.01"))
    androidTestImplementation("androidx.compose.ui:ui-test-junit4")
    androidTestImplementation("androidx.compose.ui:ui-test-manifest")
}
```

Jetpack Compose 组件数量众多，如果由开发者手动声明每个组件的版本号，容易出现冲突；因此官方提供了 BOM 文件，当我们引入该文件后，声明其他组件时就不必再指明版本号了。 BOM 文件版本与 Compose 组件版本的对应关系可参考相关网页： [🔗 Jetpack Compose BOM版本](https://developer.android.com/develop/ui/compose/bom/bom-mapping) 。

Jetpack Compose 的核心组件是必选的；调试工具是可选的，它们能够在开发阶段为开发者提供预览等功能；测试工具则提供了对于 JUnit 的支持，如果我们不需要编写测试用例，可以不声明这些依赖项。

第二步，在模块的 Gradle 配置文件中添加 Jetpack Compose 的相关配置。

`build.gradle` :

```groovy
android {
    buildFeatures {
        // 开启Compose
        compose = true
    }

    composeOptions {
        // Jetpack Compose注解处理器版本
        kotlinCompilerExtensionVersion = '1.5.7'
    }
}
```

上述内容也可以使用 Kotlin 语言编写：

`build.gradle.kts` :

```kotlin
android {
    buildFeatures {
        // 开启Compose
        compose = true
    }

    composeOptions {
        // Jetpack Compose注解处理器版本
        kotlinCompilerExtensionVersion = "1.5.7"
    }
}
```

我们首先需要在 `buildFeatures {}` 小节中开启 Jetpack Compose 编译功能，然后在 `composeOptions {}` 小节中指明编译器的版本号。编译器版本应与当前模块的 Kotlin 版本相匹配，对应关系可在 [🔗 Compose编译器版本](https://developer.android.com/jetpack/androidx/releases/compose-kotlin#pre-release_kotlin_compatibility) 网页中查询。

第三步，创建测试 Activity ，声明一些 Compose 组件，并将它们作为 Activity 的显示内容。

`TestUIBase.kt` :

```kotlin
class TestUIBase : ComponentActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        // 显示 Compose UI 组件
        setContent {
            // 应用主题
            TestComposeTheme {
                // 应用布局
                Scaffold(modifier = Modifier.fillMaxSize()) { innerPadding ->
                    // 放置控件
                    Greeting(
                        name = "Android",
                        modifier = Modifier.padding(innerPadding)
                    )
                }
            }
        }
    }

    // 声明 Compose 组件
    @Composable
    fun Greeting(name: String, modifier: Modifier = Modifier) {
        // 声明文本框
        Text(
            text = "Hello $name!",
            modifier = modifier
        )
    }

    // 声明 Compose 组件预览
    @Preview(showBackground = true)
    @Composable
    fun GreetingPreview() {
        TestComposeTheme {
            Greeting("Android")
        }
    }
}
```

在上述代码中， `Greeting()` 方法被 `@Composable` 注解修饰，因此这是一个 Compose 组件；其中的 `Text()` 方法声明了一个文本框，部分内容由 `Greeting()` 方法的第一参数 `name` 决定，以提供动态设置显示内容的能力。

`GreetingPreview()` 方法被 `@Preview` 注解修饰，在开发阶段 Android Studio 能够提供组件外观预览，在运行阶段则无作用。

在 Activity 的生命周期方法 `onCreate()` 中，我们使用 `setContent()` 方法设置界面内容；最外层的 TestComposeTheme 是主题、较内层的 Scaffold 是根布局、最内层的 Greeting 即自定义Compose组件。

## 新版本
自从 Jetpack Compose 1.6.8 （或 BOM 版本号 `2024.06.00` ）开始， Compose 编译器变更为一个独立的 Gradle 插件，且版本号跟随工程的 Kotlin 版本、不再通过 `composeOptions {}` 小节声明。

`build.gradle.kts` :

```kotlin
plugins {
    // Kotlin-Android 插件的版本号既工程的 Kotlin 版本号。
    id("org.jetbrains.kotlin.android").version("2.4.10").apply(false)
    // Compose 编译器插件，版本号跟随 Kotlin 版本。
    id("org.jetbrains.kotlin.plugin.compose").version("2.4.10").apply(false)
}
```


# 数据交互
在前文“示例一”中，我们了解了Compose的基本使用方法，但只有界面没有功能，我们还需要建立ui和viewmodel的联系，ui监听数据进行展示，下发指令进行业务接口调用。


示例二：计数器。

在本示例中，我们通过Compose实现一个计数器，初始数值为 `0` ，用户每次点击按钮，当前数值自增 `1` 。


"MyViewModel.kt":

```kotlin
class MyViewModel : ViewModel() {

    // 可变状态Flow，仅内部使用。
    private val _state: MutableStateFlow<Int> = MutableStateFlow(0)

    // 不可变状态Flow，与可变实例保持同步，供外部观察。
    val state: StateFlow<Int> = _state


    // 业务接口：数值增加。
    fun add() {
        _state.value++
    }

    // 业务接口：数值减少。
    fun reduce() {
        _state.value--
    }
}
```

## 案例一
### 问题描述
APP模块编译失败，错误信息为： `<组件> depend on it to compile against version 34 or later of the Android APIs` 。

```text
22 issues were found when checking AAR metadata:

1.  Dependency 'androidx.compose.material3:material3-android:1.2.1' requires libraries and applications that depend on it to compile against version 34 or later of the Android APIs.

# 此处省略部分输出内容...
```

### 问题分析
部分组件的较新版本不兼容API 33及更低版本的SDK，如果我们无法升级工程的编译SDK版本，则只能使用这些组件的较旧版本。

### 解决方案
目前各个组件对API 33及更高版本的支持情况尚无官方文档可供查询，经过测试，当工程的API版本为33时，各个组件能够支持的最高版本如下文列表所示：

- `androidx.compose:compose-bom` : `2023.06.01`
- `androidx.activity:activity-compose` : `1.6.1`
