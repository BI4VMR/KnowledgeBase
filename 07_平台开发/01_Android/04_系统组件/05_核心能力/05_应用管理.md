# 基本应用
## 软件包信息
PackageInfo表示软件包的基本信息，例如：包名、版本号、权限列表等，常用属性详见下文列表：

- `packageName` : 包名。
- `versionCode` : 版本号（数字）。
- `versionName` : 版本号（文本）。
- `firstInstallTime` : 首次安装时间。
- `lastUpdateTime` : 最近更新时间。
- `applicationInfo` : ApplicationInfo实例，详见后文章节。
- `activities` : 所有Activity信息。获取时需设置 `PackageManager.GET_ACTIVITIES` 标志位，否则为空值。
- `receivers` : 所有BroadcastReceiver信息。获取时需设置 `PackageManager.GET_RECEIVERS` 标志位，否则为空值。
- `services` : 所有Service信息。获取时需设置 `PackageManager.GET_SERVICES` 标志位，否则为空值。
- `providers` : 所有ContentProvider信息。获取时需设置 `PackageManager.GET_PROVIDERS` 标志位，否则为空值。
- `requestedPermissions` : 权限列表。获取时需设置 `PackageManager.GET_PERMISSIONS` 标志位，否则为空值。

下文示例展示了获取软件包基本信息的方法。

🔴 示例一：获取软件包信息。

在本示例中，我们通过PackageManager获取当前示例应用的基本信息。

"TestUIBase.java":

```java
// 通过Context获取PackageManager实例
PackageManager packageManager = this.getPackageManager();
try {
    // 使用当前软件包作为目标
    String target = getPackageName();
    // 获取该软件包的PackageInfo实例
    PackageInfo info = packageManager.getPackageInfo(target, 0);

    // API >= 33 时应当使用PackageInfoFlags表示标志位
    // PackageInfo info = packageManager.getPackageInfo(target, PackageManager.PackageInfoFlags.of(0L));

    // 获取版本号
    int versionCode = info.versionCode;
    Log.i(TAG, "数字版本号：[" + versionCode + "]");
} catch (PackageManager.NameNotFoundException e) {
    // 应用未安装
    Log.e(TAG, "App not found!");
}
```

我们首先通过Context实例获取PackageManager实例，然后调用它的 `getPackageInfo(String packageName, int flags)` 方法获取软件包信息，第一参数 `packageName` 即目标软件包的包名，第二参数 `flags` 是配置标志位，用于控制需要获取的额外信息。

解析所有Activity等组件会导致查询时间过长，因此 `getPackageInfo()` 方法默认只返回基本信息，额外信息对应的属性均为空值；若要解析额外信息，我们可以查看源码注释确定它们的Flag值，并在 `getPackageInfo()` 方法的第二参数中指明。

自从Android 13开始，我们应当使用PackageInfoFlags类表示标志位，它的静态方法 `of(int value)` 可以将数值形式的标志位组合转为PackageInfoFlags实例。为了提高兼容性，后文示例默认都使用数值形式的Flag组合。

若目标软件包未被安装在设备上， `getPackageInfo()` 以及类似的方法都会抛出 `PackageManager.NameNotFoundException` 异常。有时我们只想判断某个应用程序是否已被安装，也可以调用 `getPackageInfo()` 方法并检查是否出现异常。

## 应用信息
ApplicationInfo表示应用程序Manifest文件中 `<application>` 标签的属性，以及其他应用级别的信息，常用属性与方法详见下文列表：

- `packageName` : 包名。
- `enabled` : 是否启用。如果应用程序被禁用，则无法被外部启动，Launcher也不会显示入口图标。
- `uid` : UID。关于多用户的知识详见相关章节： [🧭 用户管理](06_用户管理.md) 。
- `minSDK` : 最低SDK版本。
- `targetSdkVersion` : 目标SDK版本。
- `compileSdkVersion` : 编译SDK版本。自从Android 12 (API 31)开始提供。
- `dataDir` : 数据目录。
- `sourceDir` : 程序目录。
- `nativeLibraryDir` : 原生库目录。
- `loadLabel(PackageManager pm)` : 应用名称。
- `loadIcon(PackageManager pm)` : 带有工作区、通知角标等徽章的图标。
- `loadUnbadgedIcon(PackageManager pm)` : Manifest文件中定义的原始图标，无徽章。

`loadIcon()` 方法返回的图标包含徽章，例如：工作区应用、通知角标等，如果我们想要获取应用程序Manifest文件中定义的原始图标，可以使用 `loadUnbadgedIcon(PackageManager pm)` 方法。

下文示例展示了获取软件包Application信息的方法。

🔴 示例二：获取Application信息。

在本示例中，我们通过PackageManager获取当前示例应用的Application信息。

"TestUIBase.java":

```java
PackageManager packageManager = this.getPackageManager();
try {
    // 使用当前软件包作为目标
    String target = getPackageName();
    // 获取该软件包的ApplicationInfo实例
    ApplicationInfo info = packageManager.getApplicationInfo(target, 0);

    // 当前是否启用

    Log.i(TAG, "是否启用：[" + enable + "]");
} catch (PackageManager.NameNotFoundException e) {
    Log.e(TAG, "App not found!");
}
```

## 组件信息
PackageManager提供了以下方法便于我们获取应用程序的Activity、Service等组件信息：

- `ActivityInfo getActivityInfo(@NonNull ComponentName component, int flags)` : 获取指定的Activity信息。
- `ServiceInfo getServiceInfo(@NonNull ComponentName component, int flags)` : 获取指定的Service信息。
- `ProviderInfo getProviderInfo(@NonNull ComponentName component, int flags)` : 获取指定的ContentProvider信息。
- `ActivityInfo getReceiverInfo(@NonNull ComponentName component, int flags)` : 获取指定的BroadcastReceiver信息。

由于BroadcastReceiver的属性较少，因此 `getReceiverInfo()` 方法的返回值复用了ActivityInfo类。

ActivityInfo的常用属性与方法详见下文列表：

- `packageName` : 包名。
- `name` : 类名。
- `targetActivity` : 目标Activity。若当前ActivityInfo是一个Activity别名，该属性提供了别名指向的实际Activity；若ActivityInfo不是别名，该属性为空值。
- `applicationInfo` : 所属应用的ApplicationInfo实例。
- `loadLabel(PackageManager pm)` : 应用名称。
- `loadIcon(PackageManager pm)` : 带有工作区、通知角标等徽章的图标。
- `loadUnbadgedIcon(PackageManager pm)` : Manifest文件中定义的原始图标，无徽章。

对于Activity等组件，相关方法优先返回组件标签中定义的名称与图标；若我们没有为它们定义名称或图标，这些方法将会返回Application标签中定义的名称或图标。

ServiceInfo、ProviderInfo与ActivityInfo类似，此处不再赘述它们的具体属性，我们可以查阅它们的源码了解详情。

下文示例展示了获取软件包组件信息的方法。

🔴 示例三：获取Activity信息。

在本示例中，我们通过PackageManager获取当前示例Activity的相关信息。

"TestUIBase.java":

```java
PackageManager packageManager = this.getPackageManager();
try {
    // 指定当前示例Activity为目标
    ComponentName cn = new ComponentName(this, getClass());
    // 获取该Component的ActivityInfo实例
    ActivityInfo info = packageManager.getActivityInfo(cn, 0);

    // Activity名称（如果Activity未定义Label属性，则返回Application中定义的名称。）

    Log.i(TAG, "标签：[" + label + "]");
} catch (PackageManager.NameNotFoundException e) {
    Log.e(TAG, "App not found!");
}
```

## 查询所有软件包

系统提供了以下API用于查询所有应用。

- `List<PackageInfo> getInstalledPackages(int flags)` : 查询所有已安装应用程序，返回PackageInfo列表。
- `List<ApplicationInfo> getInstalledApplications(int flags)` : 查询所有已安装应用程序，返回ApplicationInfo列表。

这些接口要求当前应用具有 `android.permission.QUERY_ALL_PACKAGES` 权限，并且可能耗时较长，应当在子线程调用。

## 查询特定组件


有时我们需要根据特定条件检索软件包中的组件，例如：“查询所有媒体服务”、“查询能够打开视频文件的应用”等，此时可以使用 `query`系列方法

`List<ResolveInfo> queryIntentActivities(Intent, intent, int flags)` : 查询所有与该Intent匹配的Activity。
`List<ResolveInfo> queryIntentServices(Intent, intent, int flags)` : 查询所有与该Intent匹配的Service。
`List<ResolveInfo> queryBroadcastReceivers(Intent, intent, int flags)` : 查询所有与该Intent匹配的Receiver。

🔴 示例四：查询所有MediaSession服务。

在本示例中，我们通过PackageManager查询所有支持MediaSession的服务信息。

"TestUIBase.java":

```java
PackageManager packageManager = this.getPackageManager();

// MediaSession服务的Intent
Intent intent = new Intent();
intent.setAction(MediaBrowserService.SERVICE_INTERFACE);

// 查询所有的MediaSession服务
List<ResolveInfo> resolveInfos = packageManager.queryIntentServices(intent, 0);
resolveInfos.forEach(info -> {
    // 获取ServiceInfo
    ServiceInfo serviceInfo = info.serviceInfo;
    Log.i(TAG, "Service:[" + serviceInfo.name + "]");
});
```




## 解析APK信息

​ 前面介绍的方法，都是基于一个已安装的 App 的包名，来获取额外的信息。

​ 但是有时候，我们只有一个未安装的 Apk 文件，想要解析出 Apk 文件中的额外信息，PM 中，也有对应的 Api。非常的方便，直接使用 getPackageArchiveInfo() 即可。

    /**
     * 通过Apk路径，获取Apk信息
     */
    public PackageInfo getPackageArchiveInfo (Context context, String apkPath){
        try{
            PackageManager pm = context.getPackageManager();
            PackageInfo packageInfo = pm.getPackageArchiveInfo(apkPath,PackageManager.GET_ACTIVITIES);
            return packageInfo;
        }catch(Throwable ignore){
            return null;
        }
    }

只要拿到这个 Apk 文件相关的 PackageInfo 信息，就有办法拿到其他的名称、icon 、版本号、版本名、包名等信息。和前面介绍的例子类似，这里就不再一一列举了。

# 元数据
在日常的Android开发中，AndroidManifest

中总会出现一些<meta-data>标签，或是第三方SDK配置信息，或是系统配置，不禁让人思考<meta-data>到底是什么？有什么用？怎么用？有哪些应用场景？
meta-data简介

<meta-data>
直译为“元数据”，该标签可为<activity>、<activity-alias>、<application>、<provider>、<receiver>、<service>等组件提供附加数据项。
组件元素可以包含任意数量的<meta-data>子元素。系统将meta-data配置的数据存储于一个Bundle对象中，可以通过PackageItemInfo.metaData字段获取。

ApplicationInfo appInfo = getPackageManager().getApplicationInfo(getPackageName(),
                PackageManager.GET_META_DATA);
        value = appInfo.metaData.getString("meta_app");


ActivityInfo activityInfo = getPackageManager().getActivityInfo(getComponentName(),
                PackageManager.GET_META_DATA);
        resId= activityInfo.metaData.getInt("meta_act");



语法配置

<meta-data android:name="string"
           android:resource="resource specification"
           android:value="string" />

android:name

    分配给该标签的键，即唯一名称。为确保名称唯一应使用Java风格命名约定，如“ com.example.project.activity.fred”。

android:resource

    对资源的引用，如“@string/app_name”。该资源ID可以通过该metaData.getInt()方法获取 。

android:value

    分配给该标签的值，如String、Boolean等。

meta-data用法

正如前文所说，<meta-data>可用于常见的组件，那么在程序中应该如何配置，并获取这些配置好的外部数据呢？

配置示例：

<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.fedming.notches">

    <application
        ...
        <activity android:name=".MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
            <meta-data
                android:name="meta_act"
                android:resource="@string/app_name" />
        </activity>
        <meta-data
            android:name="meta_app"
            android:value="value from meta_app" />
    </application>

</manifest>





# 调试命令
## 语法
我们可以通过ADB Shell管理软件包，以便进行调试与测试。

下文将介绍常用的软件包管理命令。

🔶 查询软件包列表

以下命令可以列出设备中安装的软件包列表：

```text
ADB$ pm list packages [选项] [--user <用户ID>]
```

当我们省略选项与用户ID时，该命令将输出所有用户的软件包名称。

该命令的常用选项如下文列表所示：

- `-s` : 筛选系统软件。
- `-3` : 筛选非系统软件。
- `-d` : 筛选已被禁用的软件。
- `-e` : 筛选已被启用的软件。
- `-i` : 显示安装来源。
- `-f` : 显示安装位置。

🔶 查询软件包的安装位置

以下命令可以列出指定软件包的安装位置：

```text
ADB$ pm path [--user <用户ID>] <软件包名称>
```

如果软件包有预装的版本和升级安装的版本，该命令将会输出多条结果。

🔶 查询软件包详情

以下命令可以输出软件包的详细信息：

```text
ADB$ dumpsys package [软件包名称]

ADB$ pm dump <软件包名称>
```

`dumpsys package` 命令能够输出软件包的基本信息，主要包括下文列表中的内容：

- 版本号 : `versionCode=<数字版本号>` 、 `versionName=<文本版本号>` 。
- 签名摘要 : `signatures=<签名信息>` 。
- UID分配情况 : `userId=<UID>` 。
- SDK版本信息 : `minSdk=<版本号>` 、 `targetSdk=<版本号>` 。
- 数据目录 : `dataDir=<目录>` 、 `codePath=<目录>` 、 `resourcePath=<目录>` 。
- 权限授予情况 : `requested permissions:` 、 `install permissions:` 和 `runtime permissions:` 。

`pm dump` 命令能够输出软件包的详细信息，包括组件信息、资源使用情况等。

🔶 查询系统支持的特性

以下命令可以查询当前设备支持的特性：

```text
# 列出所有特性
ADB$ pm list features

# 判断当前设备是否支持某种特性
ADB$ pm has-feature <特性名称>
```

## 变更一
### 摘要
自从Android 11开始，应用程序需要声明软件包可见性，才能查询其他应用程序的信息。

### 详情
自从Android 11开始，如果应用程序需要查询其他应用程序的信息，则需要在Manifest文件中声明目标程序的包名，否则会出现 `NameNotFoundException` 等异常。

我们可以通过以下配置项声明需要查询的应用程序包名：

"AndroidManifest.xml":

```xml
<manifest>
    <queries>
        <package android:name="<目标软件包的包名1>" />
        <package android:name="<目标软件包的包名2>" />
    </queries>
</manifest>
```

如果我们需要查询所有软件包的信息，则可以声明以下权限：

"AndroidManifest.xml":

```xml
<manifest>
    <!-- 查询所有软件包的信息  -->
    <uses-permission android:name="android.permission.QUERY_ALL_PACKAGES" />
</manifest>
```

关于软件包可见性的详细信息，可以参考以下链接：

- [🔗 Android官方文档 - 软件包可见性](https://developer.android.com/training/package-visibility)
