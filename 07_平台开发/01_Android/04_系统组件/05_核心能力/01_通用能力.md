## 内存信息
我们可以通过ActivityManager的 `getMemoryInfo(MemoryInfo info)` 方法获取设备内存总量、当前剩余内存等信息，当我们调用该方法时，它会将最新信息填入我们传入的MemoryInfo实例，然后我们可以通过其中的属性获取相关信息：

- `long totalMem` : 内存总量（单位：字节）。
- `long availMem` : 当前剩余内存（单位：字节）。
- `boolean lowMemory` : 当前剩余内存是否较低。如果该值为 `true` ，系统可能开始杀死后台进程以回收资源。
- `long threshold` : 剩余内存较低的阈值（单位：字节）。

# 系统兼容性
不同版本的Android提供的接口存在差异，部分接口在新旧版本的行为不同，部分用户可能仍使用旧版系统的设备，我们可以对系统版本进行判断，选择相应的业务代码实现功能。


"TestUICompatibility.java":

```java
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
    /* 系统版本高于Android 13 */
    Log.i(TAG, "系统版本高于Android 13，调用新的接口。");
    getPackageManager().getInstalledPackages(PackageManager.PackageInfoFlags.of(0L));
} else {
    /* 系统版本低于Android 13 */
    Log.i(TAG, "系统版本低于Android 13，调用旧的接口。");
    getPackageManager().getInstalledPackages(0);
}
```


各个系统版本的变更详情可在 [🔗 Android - 版本变更](https://developer.android.com/about/versions) 查询。

# 隐藏接口
Android SDK中的部分接口是给系统组件使用的，不对普通应用开放，这些方法在源码中通常具有以下注解。

`@hide` : 文档注解，表示对普通应用不可见。
`@SystemApi` : 方法注解
`@PrivateApi` : 方法注解
`@UnsupportedAppUsage` : 属性、方法、类注解，表示该元素不对普通应用开放。

编译SDK时这些属性、方法会被删除，因此我们无法通过标准的SDK直接引用它们。


当我们开发系统应用时，可以通过以下方法调用隐藏接口：


1. 下载AOSP源码，移除隐藏方法相关注解，编译生成新的 `framework.jar` 并在当前工程中引用。
2. 将隐藏方法所在的源码文件复制到当前工程对应包中，确保编译通过。
3. 通过反射机制获取变量、方法等。

其中方法一只适合定制化的产品开发，因为每个工程只能导入一个`framework.jar`文件，而且操作比较繁琐。方法二只对独立的Hide类有效，如果类中部分成员为Hide，另外部分成员非Hide，则无法实施，适用场景较少。

方法三是比较实用的，能够适应不同的系统版本，且无需繁琐的操作。
