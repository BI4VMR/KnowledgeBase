# 简介
本地广播是一种作用域在程序内部的事件总线，这种广播不会发送到程序外部。如果部分广播携带的数据不便于暴露给其它程序，又希望实现应用内的全局事件通告，此时就可以使用本地广播。

# 基本应用
## 发送消息
发送本地广播时，需要使用LocalBroadcastManager的 `sendBroadcast()` 方法。LocalBroadcastManager是一个单例类，需要使用 `getInstance()` 获取它的实例，然后进行操作。

```java
Intent intent = new Intent("net.bi4vmr.MY_BROADCAST");
LocalBroadcastManager.getInstance().sendBroadcast(intent);
```

## 接收消息
本地广播的接收方法与全局广播类似，由LocalBroadcastManager进行管理，实现广播监听器的注册与注销等操作。

```java
// 注册广播接收器
IntentFilter filter = new IntentFilter("net.bi4vmr.MY_BROADCAST");
MyReceiver receiver = new MyReceiver();
LocalBroadcastManager.getInstance(this).registerReceiver(receiver, filter);
```

注册完成后，就可以接收本应用中其它组件所发送的广播消息，并通过MyReceiver进行业务操作。

# 版本变更
## [Android 8]“本地广播”已被弃用
本地广播是应用级的事件总线，应用中的任何组件都可以监听其它组件的事件，这种设计是不好的，因此该库已经被弃用，建议使用LiveData等观察者模式实现进行替换。
