3.3.1   概述
三层交换机是具有部分网络层功能的交换机，它工作在OSI参考模型的第三层。三层交换机最重要的功能是加快大型局域网内部的数据交换，其路由能力也是为这一目的服务的，因此不适合连接异构网络。
三层交换机执行数据包转发等动作时，可使用专用芯片完成，而路由信息更新、路由表维护、路由计算等功能，则需要由软件实现。

            3.3.2   构建三层链路
三层交换机的接口不仅可以工作在数据链路层，还可以工作在网络层，通过三层链路与路由器、防火墙等设备互联。
                • 典型方式
大多数三层交换机可以将端口切换为三层模式，此时端口具有路由器接口的功能：
    • 将端口切换为三层模式
Cisco(config-if)#no switchport
    • 将端口切换为二层模式
Cisco(config-if)#switchport
                • 非典型方式
部分三层交换机不支持直接将物理端口切换为三层模式，此时我们可以创建一个专门的VLAN，把端口划入其中，然后通过SVI实现网络层功能。
1.创建VLAN：
Cisco(config)#vlan [VLAN ID]
Cisco(config-vlan)#exit
2.分配物理接口：
Cisco(config)#int [端口ID]
Cisco(config-if)#switchport mode access
Cisco(config-if)#switchport access vlan [VLAN ID]
Cisco(config-if)#exit
3.创建SVI并进行网络层配置：
Cisco(config)#int vlan [VLAN ID]
Cisco(config-if)#ip address [IP地址] [子网掩码]
Cisco(config)#exit
            3.3.3   工作流程
三层交换机的路由表相当于路由表和ARP表的组合，用来实现高速交换，其路由表的生成方式主要有两种：
    • 按需生成
交换机首先读取IP数据包头部的目的地址，将第一个数据包进行路由，同时标记该目的地址，后续具有相同标记的流量就可以直接转发到相应端口，没有必要每次都将数据包拆包并路由，这就是所谓的“一次路由，多次交换”。当长时间没有数据刷新表项后，表项将超时并被删除。
    • 预先生成
少部分交换机只要学习到ARP条目，就会自动生成主机路由，并保持主机路由与ARP条目之间的同步。这种方式对设备硬件资源消耗会略大一些，但第一个数据包就能直接进行交换，转发性能略高。





        3.2   交换虚拟接口
            3.2.1   概述
交换虚拟接口(Switch Virtual Interface,SVI)是三层交换机用于实现VLAN间路由的逻辑接口。同一台设备上每个VLAN最多可配置一个SVI，当VLAN需要与外界进行网络层通信时，就需要配置SVI。
SVI用于连接VLAN与其它区域，所以人们也把这种接口称为三层逻辑接口，它具有路由器接口的基本特性，比如：
    • 拥有独立的MAC地址。
    • 支持大部分网络层协议（包括IP协议、路由协议等）。
通常设备的默认VLAN(VLAN 1)自带SVI且不能被删除，用于对设备进行远程管理。SVI可以同时作为管理接口和VLAN内终端的默认网关，例如常见的家用路由器。
二层交换机也可以拥有SVI接口，但仅限远程管理使用，如果手动给其它VLAN创建SVI，VLAN 1的SVI将会自动关闭。
            3.2.2   SVI启用的条件
    • SVI管理状态为启用。
    • VLAN必须存在且已激活。
    • 至少VLAN中有一个Access端口启用，或设备上有Trunk接口启用。
            3.2.3   配置方法
                • 基本配置
    • 创建/删除SVI
Cisco(config)#{no} int vlan [VLAN ID]
    • 开启/关闭SVI
Cisco(config)#int vlan [VLAN ID]
Cisco(config-if)#{no} shutdown
    • 给SVI配置IP地址
Cisco(config)#int vlan [VLAN ID]
Cisco(config-if)#ip address [IP地址] [子网掩码]
