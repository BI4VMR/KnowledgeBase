# 简介
File类位于 `java.io` 包中，用于描述文件系统中所存储的文件与目录，我们可以通过该类对文件进行管理，例如：获取文件信息、创建或删除文件等。

File实例仅表示文件路径，不涉及文件内部的数据，若要读取或修改文件内容，我们应当根据文件类型进一步构造输入或输出流并进行操作，此处省略具体描述，详见相关章节： [🧭 输入输出 - 文件流](../../03_高级特性/05_输入输出/02_文件流.md) 。


# 理论基础
## 绝对路径与相对路径
以Windows盘符或斜杠( `/` )开头的路径被称作绝对路径，例如： `C:\Test` 或 `/root/.bashrc` 。绝对路径明确地指向文件或目录，无需额外信息即可寻址。

以目录或文件名称开头的路径被称作相对路径，例如：路径 `dir/test.txt` 表示参考目录内有一个 `dir` 目录，其中有一个 `test.txt` 文件。相对路径本身是不完整的，我们不知道 `dir/test.txt` 指向 `/home/user1/dir/test.txt` 或是 `/opt/dir/test.txt` ，参考路径与相对路径拼接后才能组成完整的可寻址路径。

Java程序的参考路径为程序启动时终端所在的目录(Current Working Directory, CWD)，例如：终端在 `/home/user1` 目录中执行 `java -jar /opt/app/starter.jar` 启动程序，此时参考路径为 `/home/user1` 而不是 `/opt/app` 。为了避免用户随处启动程序导致参考路径与预期不符，我们可以在启动脚本中先将终端目录切换到程序所在目录再运行程序，或者避免在程序中使用相对路径。

我们可以在相对路径中使用当前目录符号( `.` )和上层目录符号( `..` )，例如： `./test.txt` 表示当前目录下的文本文件； `../test.txt` 则表示当前目录上层目录中的文本文件。File类不支持解析Shell中的用户目录符号( `~` )，它会被视作文件名称的一部分。

## 路径分隔符
对于Linux系统路径，我们可以直接使用字符串表示；例如：路径 `/home/user1/content.txt` 应当写作 `"/home/user1/content.txt"` 。

对于Windows系统路径，反斜杠( `\` )需要使用转义符号( `\\` )表示；例如：路径 `C:\Test\content.txt` 应当写作 `"C:\\Test\\content.txt"` 。

File类中的静态变量 `separator` 将在程序启动时被初始化为当前操作系统的分隔符，因此我们推荐用它拼接路径，以便程序能够跨平台运行。

```java
// 非推荐写法
File file = new File("dir/test.txt");
File file = new File("dir\\test.txt");

// 推荐写法
File file = new File("dir" + File.separator + "test.txt");
```


# 基本应用
下文示例展示了File类的基本使用方法：

🔴 示例一：获取文件的属性。

在本示例中，我们通过File实例获取文件的属性。

"TestFile.java":

```java
String prefix = "M05_Storage/C01_File/S01_Base/src/main/resources/";
File file = new File(prefix + "test.txt");

System.out.println("文件名称：" + file.getName());
System.out.println("路径：" + file.getPath());
System.out.println("绝对路径：" + file.getAbsolutePath());
System.out.println("绝对路径（标准）：" + file.getCanonicalPath());
System.out.println("文件大小：" + file.length());
System.out.println("最后修改时间：" + file.lastModified());
System.out.println("父级目录：" + file.getParent());
System.out.println("是否存在：" + file.exists());
System.out.println("是否为文件：" + file.isFile());
System.out.println("是否为目录：" + file.isDirectory());
System.out.println("是否隐藏：" + file.isHidden());
System.out.println("是否可读：" + file.canRead());
System.out.println("是否可写：" + file.canWrite());
System.out.println("是否可执行：" + file.canExecute());
```

我们可以传入字符串形式的文件或目录路径构造File对象，除此之外File类还有以下构造方法：

- `File(String parent, String child)` : 第一参数 `parent` 表示文件所在的父级目录路径，第二参数 `child` 表示文件名称，二者将组合为完整的文件路径。
- `File(File parent, String child)` : 与上一个方法含义相同，第一参数改为File实例形式。

即使文件不存在或用户无访问权限，File实例在构造阶段也不会抛出异常，我们在进行其他操作前应当通过 `exists()` 、 `canRead()` 等方法进一步判断文件的状态。

`getAbsolutePath()` 方法返回File实例构造时父级目录与目标文件组成的完整路径，如果原始路径包括 `.` 和 `..` 符号，仍会保持原样输出。 `getCanonicalPath()` 方法则会解析 `.` 与 `..` 符号并将其替换为真实的目录，当目录不存在或无权限访问时将抛出IOException异常。

🟠 示例二：创建目录与文件。

在本示例中，我们在当前模块的资源目录中创建名为 `test` 的目录，并在其中创建一个内容为空的文件。

"TestFile.java":

```java
String prefix = "M05_Storage/C01_File/S01_Base/src/main/resources/";

// 目录路径
File dirPath = new File(prefix, "test");
// 目录已存在或无写入权限等情况将返回 `false` 。
boolean result = dirPath.mkdir();
System.out.println("目录创建结果：" + result);

// 文件路径
File filePath = new File(dirPath.getPath(), "1.txt");
// 文件已存在或无写入权限等情况将返回 `false` 。
boolean result2 = filePath.createNewFile();
System.out.println("文件创建结果：" + result2);
```

`mkdir()` 方法只能在现有目录中创建新的目录，如果路径中多级目录均不存在，则无法创建目录且返回 `false` ；若要递归创建多级目录，我们可以调用 `mkdirs()` 方法。

`createNewFile()` 方法用于创建内容为空的文件，以便后续通过输出流写入内容。如果目标文件已存在，该方法不会清空现有文件，只会返回 `false` ；如果路径中部分目录不可访问，则抛出IOException异常。

🟡 示例三：修改文件名称。

在本示例中，我们将“示例二”中生成的测试文件 `1.txt` 改名为 `2.txt` 。

"TestFile.java":

```java
String prefix = "M05_Storage/C01_File/S01_Base/src/main/resources/test";

// 原文件
File fileSrc = new File(prefix, "1.txt");
// 新文件
File fileDst = new File(prefix, "2.txt");

if (fileSrc.exists() && !fileDst.exists()) {
    boolean result = fileSrc.renameTo(fileDst);
    System.out.println("文件移动结果：" + result);
} else {
    System.out.println("源文件不存在或目标文件已存在！");
}
```

上述代码执行后，测试文件 `1.txt` 将被重命名为 `2.txt` 。

`renameTo()` 方法与Shell中的 `mv` 命令逻辑相同，如果原始文件与目标文件所在目录相同，则为重命名操作；如果原始文件与目标文件所在目录不同，则为移动操作。如果我们想要移动文件到其他目录但不改变文件名称，目标路径可以只填写至目录，省略文件名称部分。

`renameTo()` 方法只支持在同一文件系统内部移动文件，对于跨文件系统的移动操作，需要采用复制方式或使用 `nio` 库中的工具。



🔷 `boolean delete()`

功能简述：

删除File的路径所指向的文件或目录。

如果目标为目录，则必须为空目录，否则将会操作失败返回"false"。若要删除非空目录，需要递归操作，由内至外移除各级文件，具体实现可参考后文示例代码。

返回值：

删除操作成功时将返回"true"；如果权限不足导致删除失败，将返回"false"。

异常情况：

- SecurityException：当前用户没有对目标路径的写入权限。

🔷 `void deleteOnExit()`

功能简述：

当JVM退出时，删除File的路径所指向的文件或目录。

该方法适用于清理每次程序结束后不再需要的临时文件。

异常情况：

- SecurityException：当前用户没有对目标路径的写入权限。

🔷 `String[] list()`

功能简述：

列出目录的所有子项名称。

返回值：

如果目录内无任何子项，返回值为空的数组；如果File指向文件，该方法将返回空值。

异常情况：

- SecurityException：当前用户没有对目标路径的读取权限。

🔷 `String[] list(FilenameFilter filter)`

功能简述：

列出目录中符合条件的子项名称。

参数列表：

- "filter"：FilenameFilter接口的实现，可以自定义过滤规则。

返回值：

如果目录内无任何子项，返回值为空的数组；如果File指向文件，该方法将返回空值。

异常情况：

- SecurityException：当前用户没有对目标路径的读取权限。

🔷 `File[] listFiles()`

功能简述：

列出目录的所有子项。

与 `list()` 方法类似，返回子项对应的File对象。

🔷 `File[] listFiles(FilenameFilter filter)`

功能简述：

列出目录的所有子项。

与 `list(FilenameFilter filter)` 方法类似，返回子项对应的File对象，并可以指定过滤条件。

## 文件系统信息
以下方法用于获取文件系统的相关信息。

🔶 `File[] listRoots()`

功能简述：

File类的静态方法，返回所有磁盘分区对应的File对象。

对于Windows系统，该方法返回所有已挂载并分配盘符的分区，包括网络驱动器。对于Linux系统，该方法只能返回根目录所在的分区，对于额外的磁盘分区，需要使用其他方法获取路径。

🔶 `long getTotalSpace()`

功能简述：

获取当前分区的总容量。

返回值：

单位为字节。若File并未指向分区或分区无法访问，则固定返回数值"0"。

🔶 `long getFreeSpace()`

功能简述：

获取当前分区的剩余容量。

返回值：

单位为字节。若File并未指向分区或分区无法访问，则固定返回数值"0"。

🔶 `long getUsableSpace()`

功能简述：

获取当前分区的剩余容量。

该方法较 `getFreeSpace()` 方法更为精确，因此推荐使用。

返回值：

单位为字节。若File并未指向分区或分区无法访问，则固定返回数值"0"。

public static void main(String[] args) throws Exception {
    File file = new File("./");
    if (file.exists() && file.isDirectory()) {
        listFiles(file);
    }
}

// 列出目录内部的文件
static void listFiles(File file) {
    // 打印当前遍历到的目录或文件信息
    System.out.println("子项名称：" + file.getName());

    // 当前项为目录时继续操作
    if (file.isDirectory()) {
        // 获取子项
        File[] subItems = file.listFiles();
        if (subItems != null) {
            // 遍历子项
            for (File item : subItems) {
                // 递归调用当前方法，直到最内层目录。
                listFiles(item);
            }
        }
    }
}

## 获取文件系统信息

```java
// 列出所有磁盘分区
File[] disks = File.listRoots();
// 遍历磁盘分区，获取容量信息。
for (File disk : disks) {
    // 打印盘符
    System.out.print("盘符：" + disk.getPath() + "\t");
    // 获取总容量
    double totalSpace = disk.getTotalSpace() * 1.0 / 1024 / 1024 / 1024;
    // 获取剩余容量
    double usableSpace = disk.getUsableSpace() * 1.0 / 1024 / 1024 / 1024;
    // 计算已用容量
    double usedSpace = totalSpace - usableSpace;
    // 计算使用率
    double useRate = (usedSpace / totalSpace) * 100;

    System.out.printf("总量：%.2f GiB\t", totalSpace);
    System.out.printf("已用：%.2f GiB\t", usedSpace);
    System.out.printf("剩余：%.2f GiB\t", usableSpace);
    System.out.printf("使用率：%.2f%%", useRate);
    System.out.println();
}
```
