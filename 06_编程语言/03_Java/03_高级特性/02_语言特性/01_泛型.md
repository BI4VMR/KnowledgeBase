# 简介
普通变量的数据类型在声明阶段就确定了，例如 `int count` 、 `String name` 等，调用者只能传入特定类型的数据。对于框架开发等场景，固定的数据类型不够灵活，框架需要提供较高的通用性，不应当限制调用者所使用的具体数据类型。

🔴 示例一：表示二维坐标。

在本示例中，我们定义一个类表示二维坐标，允许调用者传入整数、小数等多种形式的原始数据。

第一步，定义坐标类Location1，声明两个变量分别表示X轴和Y轴坐标，并提供一系列方法。

"Location1.java":

```java
public class Location1 {

    private Object x, y;

    // 设置坐标
    public void setXY(Object x, Object y) {
        this.x = x;
        this.y = y;
    }

    // 获取坐标(X)
    public Object getX() {
        return x;
    }

    // 获取坐标(Y)
    public Object getY() {
        return y;
    }
}
```

由于普通变量只能声明为一种类型，若定义为Integer就无法传入小数，若定义为Double则无法传入整数；在编写框架组件时，我们无法预知调用者将会传入何种元素，因此只能将坐标的数据类型定义为顶级父类Object。

第二步，切换到调用者视角，创建Location1实例并传入初始坐标，然后获取坐标值比较其是否与 `(100, 100)` 相等。

"TestGenerics.java":

```java
// 创建实例并设置初始坐标
Location1 location = new Location1();
location.setXY(100, 25.1082);

// 读取数据时需要判断具体的类型
Object xRaw = location.getX();
if (xRaw instanceof Integer) {
    // 将数据转换为具体类型再操作
    boolean resultX = ((Integer) xRaw).compareTo(100) == 0;
    System.out.println("x = 100? " + resultX);
} else if (xRaw instanceof Double) {
    boolean resultX = ((Double) xRaw).compareTo(100.0) == 0;
    System.out.println("x = 100? " + resultX);
} else {
    throw new IllegalArgumentException("参数类型错误！");
}

// 比较Y轴的数据时也需要判断类型，此处省略相关代码。
```

由于坐标的数据类型为Object，我们首先需要判断其具体类型，当数据类型非数字时抛出异常，仅当数据类型为数字时才能进行等值比较。

上述代码中的判断逻辑非常繁琐，降低了程序的可维护性；并且调用者可以传入任何类型的数据，直到程序运行阶段才会发现错误，效率较低。

```java
// 随便传入一些非法数据也不会报错，直到读取数据时才能发现错误。
location.setXY("这是一些文本", new BigDecimal("0.01"));
```

泛型(Generics)是Java 5引入的特性，也被称为“参数化类型”，允许开发者将变量类型作为一个参数，无需指定具体的变量类型；当调用者使用这些变量时，再指明其具体类型。

泛型提高了程序的灵活程度，同时能够保证类型安全。虽然调用者可以自由设置泛型变量的数据类型，但这些类型只能在创建对象时被指定，后续调用者访问对象时若传入其他类型的数据，编译器能够及时地检测到错误并发出提示，不会将问题遗留到运行阶段。

定义泛型的语法如下文代码块所示：

```text
// 声明包含泛型的类
class 类名<泛型类型1，泛型类型2，...> {

    // 在变量或方法中使用泛型类型 ...
    <泛型类型1> <变量名称>;
    <泛型类型2> <变量名称>;
}
```

我们需要在类或接口名称后的尖括号( `<>` )中声明泛型参数，如果类或接口具有多个泛型参数，则以逗号( `,` )作为分隔符。每个泛型参数代表一种数据类型，我们可以将它们作为变量、方法形参或返回值的类型。

泛型参数的命名规则与常量相同，下文列表展示了常用的名称：

- `T` : Type，泛指任意类型。
- `E` : Element，表示集合中的元素。
- `K` : Key，表示键值对中的键名。
- `V` : Value，表示键值对中的键值。
- `N` : Number，表示数字。

如果我们创建的类继承自含有泛型参数的类，可以直接指明父类的泛型类型，也可以将泛型加入本类的泛型参数声明列表，由本类的调用者决定具体类型：

```text
// 父类泛型 `T1` 由子类决定
class 类名<T1，T2，...> extend 父类名称<T1> {
    // 使用泛型参数...
}
```

当调用者创建泛型类的实例时，需要指明具体的类型：

```text
// 调用者需要用具体的类型替换占位符
类名<类型实参> 变量名称 = new 构造方法<类型实参>();

// 示例：声明一个元素为Integer类型的List集合
List<Integer> list = new ArrayList<Integer>();

// 示例：简化写法，等号右侧的类型参数可根据左侧接收者推断得出，此时可省略右侧的类型声明。
List<Integer> list = new ArrayList<>();
```

当我们设置泛型参数时，不能使用基本数据类型，只能使用对应的包装类。

在部分场景中，编译器可以根据上下文自动推导类型实参，此时我们可以省略创建实例时的类型实参声明，只保留尖括号即可。

JDK内置的List、Set、Map等集合就是包含泛型参数的类，它们能够容纳大量同一类型的元素，集合框架无需知晓具体的类型，因此使用泛型参数表示元素即可，具体类型由调用者在创建集合时决定。


# 基本应用
我们可以使用泛型参数表示坐标值的类型，解决前文示例中遇到的问题。

🟠 示例二：表示二维坐标（基于泛型）。

在本示例中，我们定义一个类表示二维坐标，允许调用者传入整数、小数等多种形式的原始数据。

第一步，新增Location2类，使用泛型描述X、Y轴坐标。

"Location2.java":

```java
public class Location2<T1, T2> {

    // 变量的类型由调用者决定
    private T1 x;
    private T2 y;

    // 设置坐标
    public void setXY(T1 x, T2 y) {
        this.x = x;
        this.y = y;
    }

    // 获取坐标(X)
    public T1 getX() {
        return x;
    }

    // 获取坐标(Y)
    public T2 getY() {
        return y;
    }
}
```

此时坐标变量 `x` 和 `y` 的类型是泛型，可由调用者决定具体的类型， `setXY()` 方法的参数与获取坐标方法的返回值也需要变更为对应的泛型类型。

第二步，切换到调用者视角，创建Location2实例并传入初始坐标，然后获取坐标值比较其是否与 `(100, 100)` 相等。

"TestGenerics.java":

```java
// 创建实例并设置初始坐标
Location2<Integer, Double> location = new Location2<>();
location.setXY(100, 25.1082);

// 读取数据时无需判断类型，与创建实例时的类型一致。
Integer x = location.getX();
boolean resultX = x.compareTo(100) == 0;
System.out.println("x = 100? " + resultX);

// 比较Y轴的数据时无需判断类型，此处省略相关代码。
```

当我们构建Location2实例时，已经通过泛型指明了两个方向坐标变量的数据类型，因此 `getX()` 与 `getY()` 方法也会返回对应类型的数据，无需进行额外的类型判断。

如果我们调用Location2的 `setXY()` 方法时传入类型与泛型不匹配的数据，编译器就会提示数据类型不匹配，以便我们发现错误并进行修改。

```java
// 参数类型与声明实例时的泛型类型不一致，编译阶段将会报错。
location.setXY("这是一些文本", new BigDecimal("0.01"));
```


# 泛型方法
有时我们需要在方法级别定义泛型，将其作为方法参数或返回值，此类方法被称为“泛型方法”，通常是数据转换等工具方法。

前文“示例二”中的 `T1 getX()` 、 `setXY(T1 x, T2 y)` 等都不属于泛型方法，因为它们的泛型类型来自所属的类，并非由方法调用者指定。

🟡 示例三：泛型方法。

在本示例中，我们定义一个泛型方法，将数组转换为对应类型的列表。

第一步，定义泛型方法。

"TestGenerics.java":

```java
public static <F> List<F> arrayToList(F[] array) {
    List<F> list = new ArrayList<>();
    for (F item : array) {
        list.add(item);
    }
    return list;
}
```

`arrayToList()` 方法是一个泛型方法，访问修饰符与返回值之间的 `<F>` 表示声明泛型参数 `F` ，`<F>` 的语法与类级别的泛型参数声明列表类似，若有多个泛型参数应以逗号( `,` )分隔。

第二步，调用泛型方法。

"TestGenerics.java":

```java
// 待转换的数组
Integer[] array = {1, 2, 3, 4, 5};

// 调用泛型方法，类型由参数或返回值推断得出，此处为Integer。
List<Integer> list = arrayToList(array);

System.out.println(list);
```

当我们调用泛型方法时，编译器将根据实参或返回值接收者推断泛型参数，我们无法指明泛型类型。此处的测试数组 `array` 元素类型为Integer，因此编译器推断泛型类型也为Integer。


# 类型擦除
JVM中的泛型只在编译前有效，以便IDE和编译器进行语法检查提前发现错误；当源代码被编译为字节码后，泛型类型将被替换为Object类型或首个界限类型。

🟡 示例四：类型擦除。

在本示例中，我们通过反射获取泛型变量的类型，观察类型擦除的效果。

"TestGenerics.java":

```java
// 创建实例并指定类型
Location2<Integer, Double> location = new Location2<>(100, 25.1082);
// 尝试获取变量的类型
Field[] fields = location.getClass().getDeclaredFields();
for (Field f : fields) {
    System.out.println("变量名称：" + f.getName());
    System.out.println("变量类型：" + f.getType());
}
```

此时运行示例程序，并查看控制台输出信息：

```text
变量名称：x
变量类型：class java.lang.Object
变量名称：y
变量类型：class java.lang.Object
```

根据上述输出内容可知：

即使我们通过泛型指明了变量类型，程序运行时获取到的类型仍然为Object。

类型擦除是JVM的历史遗留问题，使我们无法在运行时获取泛型参数的具体类型，为框架开发带来了挑战。我们可以通过后文内容所涉及的泛型界限与通配符改善类型擦除带来的问题，以便更好地使用泛型。


# 泛型约束
在前文示例中，我们声明了一些泛型参数，由于存在类型擦除机制，它们编译后均为Object类型，缺少更为详细的类型信息，因此我们无法访问由继承关系引入的属性与方法。

我们可以在声明泛型参数时指定约束条件，使其在运行阶段拥有具体的类型信息，以便编写框架时能够访问泛型变量的属性或方法。

🟢 示例五：泛型约束。

在本示例中，我们编写一个泛型方法，将数值型对象的数值转换为文本并返回。

第一步，编写泛型方法。

"TestGenerics.java":

```java
private static <N extends Number & Serializable> String getText(N input) {
    // `input` 的类型被限制为 `Number` 的子类，因此我们可以调用 `doubleValue()` 方法来获取数值。
    double d = input.doubleValue();
    return Double.toString(d);
}
```

此处的 `N extends Number & Serializable` 表示泛型类型 `N` 必须是Number或Number的子类，且要求实现Serializable接口，“继承自Number类”和“实现Serializable接口”是泛型参数 `N` 的两个约束条件，各个条件之间使用与( `&` )作为分隔符。

当我们为泛型参数 `N` 设置约束条件后，它在运行时也会变更为对应的类型；泛型变量 `input` 至少是Number类型，因此我们可以调用Number类提供的 `doubleValue()` 等方法。

第二步，编写测试方法。

"TestGenerics.java":

```java
// 泛型参数为Integer，是Number的子类，编译通过。
Integer integer = 100;
String text = getText(integer);
System.out.println(text);


// 泛型参数为String，非Number的子类，编译失败。
// String text2 = getText("TEXT");
```

Integer是Number的子类，因此 `100` 被传入 `getText()` 方法后不会出现编译错误，且泛型方法能够调用其父类Number提供的 `doubleValue()` 得到数值。

String不是Number的子类，因此 `TEXT` 被传入 `getText()` 方法后将会出现编译错误。


# 形变
形变用于描述原始类型与泛型类型之间的继承关系，共有三种具体情形：

- 协变(Covariance)：B是A的子类，经过某种转换方法（记作 `trans()` ）之后， `trans(B)` 仍然是 `trans(A)` 的子类。
- 逆变(Contravariance)：B是A的子类，经过某种转换方法之后， `trans(A)` 变为 `trans(B)` 的子类。
- 不变(Invariant)：B是A的子类，经过某种转换方法之后， `trans(A)` 与 `trans(B)` 没有任何继承关系。

在Java语言中，数组是“协变”的，例如：String是Object的子类，因此 `String[]` 也是 `Object[]` 的子类，我们可以将 `String[]` 数组赋值给 `Object[]` 类型的变量。

```java
// 声明String数组
String[] array = {"A", "B", "C"};

// Object是String的父类，因此可以将String数组赋值给Object数组。
Object[] array2 = array;

// 此时仍不可写入非String数据，JVM会检查传入元素是否与数组匹配，若不匹配则抛出ArrayStoreException异常。
array2[0] = 100;
```

上述代码中的变量 `array2` 是Object数组，从形式上看它能接受任意类型的元素，由于该变量指向String数组，实际上只能接受String类型的元素。若我们写出不符合逻辑的 `array2[0] = 100` 语句，编译不会报错，但运行时系统发现类型不匹配将会抛出ArrayStoreException异常。

数组的“协变”特性使类型不匹配错误直到运行阶段才会体现，降低了程序的可靠性，因此更晚出现的泛型被设计为“不变”的，泛型类之间默认没有继承关系。

```java
// 声明字符串列表
List<String> list = new ArrayList<>();

// Object是String的父类，但不能将List<String>赋值给List<Object>，编译器会提示错误。
List<Object> list2 = list;

// 因为前一条语句编译失败，可以防止我们进一步写出其他错误的代码。
// list2.add(100);
```

在上述代码中， `List<String>` 和 `List<Object>` 不存在继承关系，因此 `List<Object> list2 = list` 语句无法通过编译，这种设计能够防止我们进一步写出 `list2.add(100)` 等不符合逻辑的代码。

泛型的不变性避免了潜在的逻辑错误，但也导致泛型类缺失部分面向对象特性，难以实现部分功能，例如下文的列表合并方法：

```java
// 定义String列表
List<String> a = new ArrayList<>();
// 定义CharSequence列表
List<CharSequence> b = new ArrayList<>();

// 合并两个列表
b.addAll(a);
```

如果合并方法定义为 `addAll(Collection<E> c)` ，该功能是无法实现的，因为列表 `b` 的泛型类型为CharSequence，它的合并方法为 `addAll(List<CharSequence> c)` ；列表 `a` 是 `List<String> ` ，并非 `List<CharSequence>` 的子类， `b.addAll(a)` 语句将会出现编译错误。

为了解决上述场景遇到的问题，扩展泛型机制的能力，Java提供了通配符( `?` )，允许实现协变和逆变。
