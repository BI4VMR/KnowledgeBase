# 简介
面向对象模型明确了多个实体之间的交互关系，但在部分场景中反而增加了复杂度。例如：我们想要将列表中的元素倒序排列，按照面向对象模型需要创建Comparator接口的匿名对象，并实现 `compare()` 抽象方法设置排序规则。

🔴 示例一：将整数列表中的元素倒序排列。

在本示例中，我们定义一个整数列表，并将其中的元素倒序排列。

"TestBase.java":

```java
// 创建测试列表
List<Integer> list = new ArrayList<>();
list.add(1);
list.add(2);
list.add(3);

// 设置排序规则，将所有元素降序排列。
list.sort(new Comparator<Integer>() {

    @Override
    public int compare(Integer o1, Integer o2) {
        // 返回负数表示o1排在前面；返回正数表示o2排在前面。
        return o2.compareTo(o1);
    }
});
```

我们只想将“倒序排列”这一行为传递给 `sort()` 方法，但Java语言不支持以函数作为参数，我们需要创建Comparator接口的匿名对象并作为参数。

函数式编程的思想是将行为直接作为参数在方法间传递，更适合表达数据过滤、排序、转换等逻辑，相比面向对象风格更加简洁。


# Lambda表达式
## 简介
只含有唯一抽象方法的接口被称为函数式接口(Functional Interface)，例如前文示例中的Comparator接口，其用途就是将行为作为参数传递。

自从Java 8开始，我们可以使用Lambda表达式表示行为，将其改为函数式风格，简化代码。 Lambda表达式是一种匿名函数，可以作为方法的参数传递，用于简化只有单个抽象方法的匿名内部类的写法。

简化函数式接口的匿名内部类

我们可以将匿名内部类改为等价的lambda表达式，简化代码。

当某个方法的参数类型为函数式接口时，我们可以将匿名内部类改写为Lambda表达式的形式：

🔴 示例二：将整数列表中的元素倒序排列（使用Lambda表达式）。

在本示例中，我们定义一个整数列表，并将其中的元素倒序排列。

"TestBase.java":

```java
// 创建测试列表
List<Integer> list = new ArrayList<>();
list.add(1);
list.add(2);
list.add(3);

// 设置排序规则，将所有元素降序排列。
list.sort((o1, o2) -> o2.compareTo(o1));
```

此时 Lambda表达式 `(o1, o2) -> o2.compareTo(o1)` 等价于前文“示例一”中的匿名内部类，语法更加简洁。

---

Lambda表达式的标准格式为：

```text
(<参数1类型> <参数1名称>, <参数2类型> <参数2名称>, ...) -> {<函数体>}
```

前文“示例一”中的 `int compare(Integer o1, Integer o2)` 方法转换为Lambda表达式后记作 `(Integer o1, Integer o2) -> { return o2.compareTo(o1); }` ，该表达式还可以进一步简化：

由于Lambda表达式由函数式接口派生而来，编译器能够根据原接口的参数列表推断出参数类型，我们可以省略参数类型，记作：`(o1, o2) -> { return o2.compareTo(o1); }`

当方法体只有一条语句时，我们可以省略方法体括号与 `return` 关键字，记作：`(o1, o2) -> o2.compareTo(o1)`

有时Lambda表达式仅有一个参数，此时可以省略参数列表的括号，例如 `(String text) -> System.out.println(text)` 等价于 `text -> System.out.println(text)` 。

如果lambda仅调用单个方法，且该方法参数、返回值与lambda完全相同，整个lambda表达式可以进一步简化为方法引用。



## 引用表达式
通常情况下，Lambda表达式在作为参数传入时被直接创建和使用。有时我们希望将Lambda表达式保存为变量，以便在多处复用。Java不支持将Lambda表达式直接作为变量类型，因此需要借助函数式接口来保存对它的引用。

🟠 示例二：将线程任务保存为变量并复用。

在本示例中，我们将一个线程任务保存为Runnable类型的变量，并将其传给多个Thread对象。

"TestBase.java":

```java
// 将Lambda表达式保存为函数式接口类型的变量
Runnable task = () -> System.out.println("Thread is running. Name: " + Thread.currentThread().getName());

// 创建多个线程，复用同一个任务
Thread thread1 = new Thread(task);
Thread thread2 = new Thread(task);
thread1.start();
thread2.start();
```

## 函数式接口
除了Comparator、Runnable等具有特定含义的函数式接口，JDK的 `java.util.function` 包中还提供了一批通用接口，供框架开发者使用。

下表展示了常用的内置函数式接口：

<div align="center">

| 接口                  | 参数 | 返回值  | 描述                                            |
| :-------------------- | :--- | :------ | :---------------------------------------------- |
| `Consumer<T>`         | T    | 无      | 消费一个T类型的参数，无返回值。                 |
| `BiConsumer<T, U>`    | T, U | 无      | 消费两个参数，无返回值。                        |
| `Supplier<T>`         | 无   | T       | 无参数，返回一个T类型的值。                     |
| `Function<T, R>`      | T    | R       | 接受一个T类型的参数，返回R类型的值。            |
| `BiFunction<T, U, R>` | T, U | R       | 接受两个参数，返回R类型的值。                   |
| `Predicate<T>`        | T    | boolean | 接受一个T类型的参数，返回布尔值，用于条件判断。 |

</div>


对于我们自定义的函数式接口，应当添加 `@FunctionalInterface` 注解，便于调用者识别接口类型，并且使编译器检查接口是否符合函数式要求，若不符合定义则报错提醒开发者。

我们可以定义一个Transformer接口将一种类型转为另一种类型。

```java
@FunctionalInterface
public interface Transformer<T, R> {

    // 唯一的抽象方法
    R transform(T input);
}
```


## "this"关键字
在匿名内部类中，`this` 关键字指向该匿名内部类自身的实例；而在Lambda表达式中，`this` 关键字则指向声明该Lambda表达式的外部类实例。这是Lambda表达式与匿名内部类的一个重要区别。

🔵 示例五：比较Lambda与匿名内部类中this的指向。

在本示例中，我们分别在Lambda表达式和匿名内部类中使用 `this` 关键字，观察它们的指向差异。

"TestBase.java":

```java
public class TestBase {

    private String name = "TestBase";

    public void test() {
        // Lambda中的this指向外部类（TestBase）的实例
        Runnable lambda = () -> System.out.println("Lambda this: " + this.name);

        // 匿名内部类中的this指向匿名内部类自身的实例
        Runnable anonymous = new Runnable() {
            private String name = "AnonymousRunnable";

            @Override
            public void run() {
                System.out.println("Anonymous this: " + this.name);
            }
        };

        lambda.run();
        anonymous.run();
    }
}
```

此时运行示例程序，并查看控制台输出信息：

```text
Lambda this: TestBase
Anonymous this: AnonymousRunnable
```
