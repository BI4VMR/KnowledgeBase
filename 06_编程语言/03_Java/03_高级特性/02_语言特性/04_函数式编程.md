# 简介
面向对象模型明确了实体之间的交互关系，能够更好地组织大型项目的结构，但在部分场景中反而提高了复杂度。例如：我们希望将列表中的元素倒序排列，需要创建Comparator接口的匿名对象，并实现 `compare()` 抽象方法设置排序规则。

🔴 示例一：将整数列表中的元素倒序排列。

在本示例中，我们定义一个整数列表，并将其中的元素倒序排列。

"TestBase.java":

```java
// 创建测试列表
List<Integer> list = new ArrayList<>(Arrays.asList(1, 2, 3, 4, 5));

// 设置排序规则，将所有元素降序排列。
list.sort(new Comparator<Integer>() {

    @Override
    public int compare(Integer o1, Integer o2) {
        // 返回当前两个数值的比较结果
        return o2.compareTo(o1);
    }
});
```

我们只想将“倒序排列”这一行为传递给 `sort()` 方法，但Java语言不支持以函数作为参数，我们不得不创建Comparator接口的匿名对象作为参数，语法较为繁琐。

Comparator等接口仅含有唯一的抽象方法，调用者创建它们的实例就是为了重写抽象方法传递行为，此类接口被称为函数式接口(Functional Interface)。

函数式编程的思想旨在将行为直接作为参数在方法间传递，无需创建匿名对象，适合表达数据过滤、排序、转换等场景的逻辑，相比面向对象语法更加简洁。


# Lambda表达式
## 简介
Lambda表达式是一种匿名函数，我们可以将函数式接口的匿名内部类替换为等价的Lambda表达式，简化代码。自从Java 8开始，我们可以使用Lambda表达式进行函数式编程。

以Comparator接口为例，我们对前文“示例一”进行修改，利用Lambda表达式设置排序规则。

🟠 示例二：将整数列表中的元素倒序排列（使用Lambda表达式）。

在本示例中，我们定义一个整数列表，并将其中的元素倒序排列。

"TestBase.java":

```java
// 创建测试列表
List<Integer> list = new ArrayList<>(Arrays.asList(1, 2, 3, 4, 5));

// 设置排序规则，将所有元素降序排列。
list.sort((o1, o2) -> o2.compareTo(o1));
```

此时Lambda表达式 `(o1, o2) -> o2.compareTo(o1)` 等价于前文“示例一”中的匿名内部类，语法更加简洁。

## 语法
Lambda表达式的标准格式为：

```text
(<参数1类型> <参数1名称>, <参数2类型> <参数2名称>, ...) -> {<函数体>}
```

前文“示例一”中的 `int compare(Integer o1, Integer o2)` 方法转换为Lambda表达式后记作 `(Integer o1, Integer o2) -> { return o2.compareTo(o1); }` ，该表达式还可以进一步简化。

该表达式由函数式接口转换而来，编译器能够根据抽象方法的参数列表推断参数类型，因此我们可以省略参数类型，记作： `(o1, o2) -> { return o2.compareTo(o1); }` ；当函数体只有单条语句时，我们可以省略函数体括号与 `return` 关键字，记作： `(o1, o2) -> o2.compareTo(o1)` 。

除了上述示例中的情况，Lambda表达式还有其他简化规则。

当Lambda表达式仅有单个参数时，我们可以省略参数列表的括号，例如 `(String text) -> System.out.println(text)` 等价于 `text -> System.out.println(text)` 。

若Lambda表达式仅调用单个方法，且该方法的参数列表、返回值与表达式完全相同，整个表达式可以简化为方法引用。

🟡 示例三：方法引用。

在本示例中，我们使用方法引用简化Lambda表达式。

第一步，将排序规则提取为一个方法。

"TestBase.java":

```java
static int reverse(Integer a, Integer b) {
    return b.compareTo(a);
}
```

第二步，创建测试数据并设置排序规则。

"TestBase.java":

```java
// 创建测试列表
List<Integer> list = new ArrayList<>(Arrays.asList(1, 2, 3, 4, 5));

// 将Lambda参数传递给 `reverse()` 方法，并返回其结果。
list.sort((o1, o2) -> reverse(o1, o2));

// 此时可以简化为方法引用形式： `<类名>::<方法名称>` 。
list.sort(TestBase::reverse);
```

在上述代码中，Lambda表达式 `(o1, o2) -> reverse(o1, o2)` 的参数列表、返回值与 `reverse()` 方法一致，因此我们可以将整个表达式替换为 `TestBase::reverse` 。

该示例中的 `reverse()` 是静态方法，故方法引用的格式为 `<类名>::<方法名称>` ；如果需要引用实例方法，则语法为 `<对象名称>::<方法名称>` 或 `this::<方法名称>` 。

---

Lambda表达式不仅是函数式接口实例化的等价语法，在运行时它们的行为也存在差异。

如果Lambda表达式未引用任何外部变量，编译器只会创建单例对象，每次表达式被调用时均复用该对象，例如： `text -> System.out.println(text)` 。函数式接口实例化每次都要创建对象，在此场景中占用内存更多。

如果Lambda表达式引用了外部变量，每次表达式被调用时都要创建对象保存变量引用，例如： `instance::method` ，此时Lambda表达式的性能与函数式接口实例化一致。

## 引用表达式
有时我们希望将Lambda表达式保存在变量中，以便多次复用。Java语言不支持将函数作为变量类型，我们需要借助函数式接口保存对表达式的引用。

🟢 示例四：引用Lambda表达式。

在本示例中，我们使用Lambda表达式声明线程任务，并创建多个线程执行该任务。

"TestBase.java":

```java
// 将Lambda表达式保存为函数式接口类型的变量
Runnable task = () -> System.out.println("Thread Name: " + Thread.currentThread().getName());


// 创建多个线程，复用同一个任务
new Thread(task).start();
new Thread(task).start();
```

## 函数式接口
除了Comparator、Runnable等具有特定含义的函数式接口，JDK的 `java.util.function` 包中还提供了一批通用接口，供框架开发者使用。

<div align="center">

| 接口                  | 抽象方法                | 描述                                              |
| :-------------------- | :---------------------- | :------------------------------------------------ |
| `Consumer<T>`         | `void accept(T t)`      | 接收 `T` 类型的参数，无返回值。                   |
| `BiConsumer<T, U>`    | `void accept(T t, U u)` | 接受 `T` 和 `U` 类型的参数，无返回值。            |
| `Supplier<T>`         | `T get()`               | 无参数，返回 `T` 类型的值。                       |
| `Function<T, R>`      | `R apply(T t)`          | 接收 `T` 类型的参数，返回 `R` 类型的值。          |
| `BiFunction<T, U, R>` | `R apply(T t, U u)`     | 接受 `T` 和 `U` 类型的参数，返回 `R` 类型的值。   |
| `Predicate<T>`        | `boolean test(T t)`     | 接收 `T` 类型的参数，返回布尔值，常用于过滤逻辑。 |

</div>

我们也可以自行定义函数式接口，并在接口上声明 `@FunctionalInterface` 注解，便于调用者识别接口类型；编译器将会检查被注解的接口是否符合函数式规范，避免误用。

🔵 示例五：定义数据转换接口。

在本示例中，我们定义一个函数式接口，表示将一种类型的数据转换为另一种类型。

"TestBase.java":

```java
@FunctionalInterface
public interface Transformer<T, R> {

    // 唯一的抽象方法
    R transform(T input);
}
```

## "this"关键字
当我们使用匿名内部类或Lambda表达式实现函数式接口时，都可以使用 `this` 关键字，但两种场景中该关键字的含义有所差异。

🟣 示例六： `this` 关键字。

在本示例中，我们辨析Lambda表达式与匿名内部类中 `this` 关键字的区别。

第一步，在测试类中编写一个成员方法，使用Lambda形式和匿名类形式分别创建Runnable接口的实例。

"TestBase.java":

```java
public class TestBase {

    void testThis() {
        // 测试类实例的 `this` 引用
        System.out.println("this in test class: " + this);


        // Lambda表达式中的 `this` 指向包含它的实例，即 `TestBase` 的实例。
        Runnable lambda = () -> System.out.println("this in lambda: " + this);
        lambda.run();


        // 匿名内部类中的 `this` 指向类的实例，可以访问其属性。
        Runnable object = new Runnable() {

            // 匿名内部类可以拥有属性
            private String name = "Consumer";

            @Override
            public void run() {
                System.out.println("this in annonymous class: " + this);
                // `可以访问其属性。
                System.out.println("get name by this: " + this.name);
            }
        };
        object.run();
    }
}
```

第二步，在 `main()` 方法中创建测试类的实例，并调用该成员方法。

此时运行示例程序，并查看控制台输出信息：

```text
this in test class: net.bi4vmr.study.lambda.TestBase@251a69d7
this in lambda: net.bi4vmr.study.lambda.TestBase@251a69d7
this in annonymous class: net.bi4vmr.study.lambda.TestBase$2@8bcc55f
get name by this: Consumer
```

根据上述输出内容可知：

匿名内部类的 `this` 指向自身实例，可以访问匿名内部类的成员变量；Lambda表达式的 `this` 等价于外部类的 `this` 。

虽然Lambda表达式在JVM内部仍有实例，但函数是无状态的，不能拥有成员变量，若将 `this` 指向Lambda实例没有实际意义；因此Lambda表达式中的 `this` 仍指向外部类，不具有独立的上下文，符合函数的定义。

## 捕获外部变量
Lambda表达式可能是异步执行的，并且可以访问外部变量，如果外部变量是方法的局部变量，需要在方法执行完毕后立即回收，为了使得Lambda表达式在方法回收后还能访问外部变量，编译器需要进行一些处理，这被称为变量捕获。

🟤 示例七：捕获外部变量。

在本示例中，我们尝试在Lambda表达式中访问和修改外部变量，并分析限制条件。

"TestBase.java":

```java
public class TestBase {

    void testCapture() {
        // 定义一个局部变量
        final String s = "-";

        // 定义Lambda表达式，尝试访问和修改外部变量。
        Runnable task = () -> {
            // 允许读取外部变量
            System.out.println("读取外部变量：" + s);

            // 禁止修改外部变量，该语句无法通过编译。
            // s = "modify";
        };
    }
}
```

Lambda会将外部变量的值复制一份保存在Lambda对象中，即使外部变量随方法结束而被销毁，Lambda表达式仍能访问变量的值。

捕获具有一些限制，要求外部变量必须是final或逻辑上不可变的。此处s必须是final的，或声明后未被重新赋值的。

对于基本数据类型的变量，由于Lambda内部只是复制了它们的值，即使修改了变量值也不会同步到原始变量，这不符合逻辑。


如果我们希望Lambda表达式修改数据后外部能够感知，可以使用引用类型包装基本类型数据，例如AtomicInteger等，这样即解决了线程安全问题，又解决了基本类型无法修改的问题。
