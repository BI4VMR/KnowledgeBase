# 基础语法
在 Kotlin/JVM 中，由于 `synchronized` 和 `volatile` 不是关键字，它们通过注解或内置函数实现。

## 易变变量（Volatile）
使用 `@Volatile` 注解标记属性，可以确保该变量的**内存可见性**：对该变量的写操作会立即刷新到主内存，且其他线程读取时会直接从主内存获取最新值。

```kotlin
@Volatile
var isRunning: Boolean = true
```

> [!IMPORTANT]
> `@Volatile` 仅保证可见性，不保证原子性。对于 `i++` 等复合操作，仍需配合锁或原子类使用。

## 线程同步（Synchronized）
Kotlin 提供了两种方式实现互斥同步：

### 同步方法
使用 `@Synchronized` 注解标记函数。其效果等同于 Java 中在方法声明上添加 `synchronized` 关键字，默认使用当前实例（`this`）作为锁。

```kotlin
@Synchronized
fun increment() {
    counter++
}
```

### 同步代码块
使用内置函数 `synchronized()`。它接受一个锁对象作为参数，并在闭包内执行同步代码。

```kotlin
private val lock = Any()

fun update() {
    synchronized(lock) {
        // 临界区代码
        counter++
    }
}
```


# 实用工具
Kotlin 提供了便捷的全局方法 `thread()` 用于快速创建并配置线程。

```kotlin
public fun thread(
    start: Boolean = true,
    isDaemon: Boolean = false,
    contextClassLoader: ClassLoader? = null,
    name: String? = null,
    priority: Int = -1,
    block: () -> Unit
): Thread {
    // ...
}
```

### 参数说明
- `start` : 是否立刻启动线程。若为 `false` ，需手动调用 `start()` 。
- `isDaemon` : 是否设置为守护线程。
- `contextClassLoader` : 指定线程的类加载器。
- `name` : 设置线程名称。
- `priority` : 设置线程优先级。
- `block` : 线程执行的具体逻辑（Lambda 表达式）。

### 使用示例
```kotlin
thread(name = "MyThread") {
    println("${Thread.currentThread().name} is running")
}
```
