# 简介
继承(Inheritance)是类与类之间的关系，是一种"is-a"关系。子类继承父类之后，就拥有了父类所有的公开成员（属性和方法）。继承实现了代码的复用。

与Java不同，**Kotlin中所有的类默认都是final的**，即默认不可被继承。若要允许一个类被继承，必须使用"open"关键字修饰。同样地，若要允许方法被子类重写，也需要对方法使用"open"关键字。

# 基本应用
继承的语法如下文代码块所示：

```text
open class <父类名称> {
    open fun <可被重写的方法>() { ... }
}

class <子类名称> : <父类名称>() {
    override fun <重写的方法>() { ... }
}
```

子类使用": 父类名称()"继承父类（括号中填写传给父类构造方法的参数），并使用"override"关键字重写父类的方法。

🔴 示例一：继承的基本应用。

在本示例中，我们定义一个父类Father和子类Child，子类重写父类的 `show()` 方法。

"Father.kt":

```kotlin
open class Father {

    open fun show() {
        println("This is Father.")
    }
}
```

"Child.kt":

```kotlin
class Child : Father() {

    override fun show() {
        println("This is Child.")
    }
}
```

"TestInherit.kt":

```kotlin
val t1: Father = Father()
val t2: Father = Child()

t1.show()
t2.show()
```

此时运行示例程序，并查看控制台输出信息：

```text
This is Father.
This is Child.
```

# "super"关键字
"super"关键字用于在子类中访问父类的属性与方法，与Java用法相同。

🟠 示例二：在子类中调用父类方法。

在本示例中，子类的 `show()` 方法在调用父类方法的基础上附加了自己的逻辑。

"TestInherit.kt":

```kotlin
val child = object : Father() {
    override fun show() {
        // 调用父类的方法
        super.show()
        println("This is extended in Child.")
    }
}
child.show()
```

此时运行示例程序，并查看控制台输出信息：

```text
This is Father.
This is extended in Child.
```

# 初始化顺序
当创建子类对象时，初始化顺序为：父类主构造函数 → 父类 `init` 块 → 子类主构造函数 → 子类 `init` 块。

🟡 示例三：观察继承中的初始化顺序。

"TestInherit.kt":

```kotlin
open class Base {
    init {
        println("父类init块执行了")
    }
}

class Derived : Base() {
    init {
        println("子类init块执行了")
    }
}

// 创建子类对象
Derived()
```

此时运行示例程序，并查看控制台输出信息：

```text
父类init块执行了
子类init块执行了
```

# 防止重写
如果希望子类中某个被override的方法不再被其子类重写，可以在"override"前添加"final"关键字：

```kotlin
open class Father {
    open fun show() { println("Father") }
}

class Child : Father() {
    final override fun show() { println("Child（不可再被重写）") }
}
```
