# 简介
类包含一组功能的具体实现，而接口(Interface)只定义"标准"，即声明类必须提供哪些功能，并不关心这些功能的具体实现细节。

Kotlin的接口与Java基本相同，但有一个重要的增强：**Kotlin接口中的方法可以有默认实现**，这在Java 8之前是不支持的。

一个类可以实现多个接口，同时符合多个"标准"。

# 基本应用
定义接口的语法如下文代码块所示：

```text
interface <接口名称> {

    // 定义抽象属性
    val <属性名称>: <类型>

    // 定义抽象方法
    fun <方法名称>(): <返回值类型>

    // 定义带默认实现的方法（Kotlin独有）
    fun <方法名称>() {
        [默认实现]
    }
}
```

实现接口使用": 接口名称"语法（与继承父类相同，同时继承/实现时以逗号分隔）。

🔴 示例一：接口的基本应用。

在本示例中，我们定义Animal接口及其两个实现类，并通过接口类型操作它们。

"Animal.kt":

```kotlin
interface Animal {
    // 定义属性（带默认实现）
    val typeName: String
        get() = "Animal"

    // 定义抽象方法
    fun eat()

    // 带默认实现的方法
    fun breathe() {
        println("$typeName 在呼吸...")
    }
}
```

"Dog.kt":

```kotlin
class Dog : Animal {

    override fun eat() {
        println("类型：$typeName")
        println("狗吃狗粮")
    }
}
```

"Cat.kt":

```kotlin
class Cat : Animal {

    override fun eat() {
        println("猫吃猫粮")
    }
}
```

"TestInterface.kt":

```kotlin
// 定义集合，元素需要满足"动物"接口
val animals = mutableListOf<Animal>()

val dog = Dog()
val cat = Cat()

animals.add(dog)
animals.add(cat)

// 调用接口中定义的方法
for (animal in animals) {
    animal.eat()
}
```

此时运行示例程序，并查看控制台输出信息：

```text
类型：Animal
狗吃狗粮
猫吃猫粮
```

# 抽象类
"抽象类(Abstract Class)"是包含抽象方法的类，使用"abstract"关键字声明，不能被实例化。抽象类与接口相比，可以拥有状态（属性值）和构造函数，适合有共同属性的类层次。

与Java不同，Kotlin中的抽象类同样需要父类使用"open"关键字（或直接使用abstract修饰，abstract类默认是open的）。

我们将前文示例中的Animal接口改为抽象类：

"Animal.kt":

```kotlin
abstract class Animal {

    // 定义属性
    private val name = "Animal"

    // 定义抽象方法（子类必须实现）
    abstract fun eat()

    // 定义普通方法（子类直接继承）
    fun speak() {
        println("这是一个$name")
    }
}
```

"Dog.kt":

```kotlin
class Dog : Animal() {

    override fun eat() {
        println("狗吃狗粮")
    }
}
```

"TestAbstractClass.kt":

```kotlin
val dog = Dog()
// 调用Animal类中的普通方法
dog.speak()
// 调用Dog类实现的抽象方法
dog.eat()
```

此时运行示例程序，并查看控制台输出信息：

```text
这是一个Animal
狗吃狗粮
```

# 接口与抽象类的区别

<div align="center">

|        特性        |      接口(interface)       |   抽象类(abstract class)   |
| :----------------: | :------------------------: | :------------------------: |
|     是否可实例化    |            ❌               |             ❌              |
|   可实现/继承数量   |         多个接口           |        只能有一个父类       |
|    可以有构造函数   |            ❌               |             ✅              |
|    可以有状态属性   |   ❌（属性无幕后字段）      |             ✅              |
|    方法默认实现    |     ✅（Kotlin支持）        |             ✅              |
|      适用场景      | 能力/契约的抽象（如Runnable）| 同类事物的共性抽象（如Animal）|

</div>
