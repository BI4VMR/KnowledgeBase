# 简介
在软件设计思想中，我们将系统划分为多个模块后，每个模块的职责是明确的，仅需暴露外界所关注的接口即可，其他元素都应当隐藏起来，以防意外修改造成非预期的结果，这就是"封装(Encapsulation)"的思想。

# 类的封装
Kotlin同样使用"包(Package)"的概念管理源代码。包名格式通常为 `<反写域名>.<模块名称>` ，包声明使用"package"关键字写在文件第一行，导入其他包使用"import"关键字。

"TestEncapsulation.kt":

```kotlin
package net.bi4vmr.study.base

import java.util.UUID
import java.io.File

fun example() {
    // 调用Util包中的类
    UUID.randomUUID()
    // 调用IO包中的类
    File("/")
}
```

# 类成员的封装
## 访问修饰符
访问修饰符用来控制属性和方法的可见范围，合理使用它们可以实现良好的封装。

<div align="center">

| 访问修饰符  | 本类  | 模块内部 | 子类  | 模块外部 |
| :---------: | :---: | :------: | :---: | :------: |
|  `<默认>`   |   ✅   |    ✅     |   ✅   |    ✅     |
| `protected` |   ✅   |    ✅     |   ✅   |    ❌     |
| `internal`  |   ✅   |    ✅     |   ❌   |    ❌     |
|  `private`  |   ✅   |    ❌     |   ❌   |    ❌     |

</div>

Kotlin默认的访问修饰符是"public"（可省略），与Java的默认"包级别"不同。

## Data Class（数据类）
在Java中，用于传递数据的类（POJO/Java Bean）需要手动编写构造方法、Getter/Setter、`toString()`、`equals()`等样板代码。Kotlin提供了"数据类(data class)"，编译器会自动为所有主构造函数中声明的属性生成这些方法，大幅减少样板代码。

数据类的语法如下文代码块所示：

```text
data class <类名>(<属性列表>)
```

数据类会自动生成以下内容：

- `equals()` / `hashCode()` ：基于主构造函数中的属性进行比较。
- `toString()` ：格式为 `类名(属性1=值1, 属性2=值2, ...)` 。
- `copy()` ：创建对象的浅拷贝，可选修改部分属性。

🔴 示例一：数据类的基本应用。

在本示例中，我们使用data class定义学生类，并演示自动生成的方法。

"Student.kt":

```kotlin
import java.sql.Timestamp

data class Student(
    val id: String = "",
    var name: String = "",
    var birthDay: Timestamp? = null
)
```

"TestEncapsulation.kt":

```kotlin
val student = Student()
// 直接赋值（等同于Java的setter）
student.name = "张三"
// 直接读取（等同于Java的getter）
println("name的值为：${student.name}")
```

此时运行示例程序，并查看控制台输出信息：

```text
name的值为：张三
```

🟠 示例二：data class自动生成的方法。

在本示例中，我们演示data class自动生成的toString、equals、copy等方法。

"TestEncapsulation.kt":

```kotlin
val s1 = Student(id = "001", name = "张三")
val s2 = Student(id = "001", name = "张三")

// toString()
println("对象信息：$s1")
// equals()：比较内容而非引用（==）
println("内容比较：${s1 == s2}")
// 引用比较（===）
println("引用比较：${s1 === s2}")
// copy()：浅拷贝并可选修改部分属性
val s3 = s1.copy(name = "李四")
println("拷贝后：$s3")
```

此时运行示例程序，并查看控制台输出信息：

```text
对象信息：Student(id=001, name=张三, birthDay=null)
内容比较：true
引用比较：false
拷贝后：Student(id=001, name=李四, birthDay=null)
```

# 内部类
## 简介
Kotlin中同样支持嵌套类与内部类，但默认行为与Java不同：

- **嵌套类（默认）**：不带"inner"关键字，等同于Java的静态内部类，不持有外部类引用，无法访问外部类的非静态成员。
- **内部类**：需要添加"inner"关键字，等同于Java的成员内部类，持有外部类引用，可以访问外部类的所有成员。

## 成员内部类
成员内部类使用"inner"关键字声明，可以直接访问外部类的所有数据，包括私有属性。

在访问外部类的成员时，若名称与内部类成员有冲突，使用 `this@外部类名` 限定外部类的"this"引用。

🟡 示例三：成员内部类的基本应用。

在本示例中，我们在Outer类中定义Inner成员内部类，并在内部类中访问外部类的成员。

"Outer.kt":

```kotlin
class Outer {
    private val name = "外部类"
    val age = 20

    inner class Inner {
        val name = "内部类"

        fun show() {
            println("外部类中的name：${this@Outer.name}")
            println("内部类中的name：$name")
            println("外部类中的age：$age")
        }
    }
}
```

"TestEncapsulation.kt":

```kotlin
// 创建外部类的对象
val outer = Outer()
// 创建内部类的对象
val inner = outer.Inner()
// 调用内部类对象的show方法
inner.show()
```

此时运行示例程序，并查看控制台输出信息：

```text
外部类中的name：外部类
内部类中的name：内部类
外部类中的age：20
```

## 匿名对象
Kotlin使用"object : 接口/父类"语法创建匿名对象，对应Java的匿名内部类。相比Java，Kotlin还可以通过Lambda表达式更简洁地替代单方法接口的匿名对象。

🟢 示例四：匿名对象的基本应用。

在本示例中，我们使用匿名对象实现USB状态回调接口。

"TestEncapsulation.kt":

```kotlin
// 使用匿名对象实现接口
USBManager.addUSBStateCallback(object : USBManager.USBStateCallback {
    override fun onPlugged() {
        println("检测到USB设备插入！")
    }

    override fun onUnPlugged() {
        println("检测到USB设备拔出！")
    }
})
```
