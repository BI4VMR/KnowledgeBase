# 简介
面向对象模型明确了实体之间的交互关系，能够更好地组织大型项目的结构，但在部分场景中反而提高了复杂度。例如：我们希望将列表中的元素倒序排列，需要创建Comparator接口的匿名对象，并实现 `compare()` 抽象方法设置排序规则。

🔴 示例一：将整数列表中的元素倒序排列。

在本示例中，我们定义一个整数列表，并将其中的元素倒序排列。

"TestBase.kt":

```kotlin
// 创建测试列表
val list = mutableListOf(1, 2, 3, 4, 5)

// 设置排序规则，将所有元素降序排列。
list.sortWith(object : Comparator<Int> {

    override fun compare(o1: Int, o2: Int): Int {
        // 返回当前两个数值的比较结果
        return o2.compareTo(o1)
    }
})
```

我们只想将“倒序排列”这一行为传递给 `sort()` 方法，按照面向对象的语法风格，我们需要创建Comparator接口的匿名对象作为参数，语法较为繁琐。

Comparator等接口仅含有唯一的抽象方法，调用者创建它们的实例就是为了重写抽象方法传递行为，此类接口被称为函数式接口(Functional Interface)。

函数式编程的思想旨在将行为直接作为参数在方法间传递，无需创建匿名对象，适合表达数据过滤、排序、转换等场景的逻辑，相比面向对象语法更加简洁。


# Lambda表达式
## 简介
Lambda表达式是一种匿名函数，我们可以将函数式接口的匿名内部类替换为等价的Lambda表达式，简化代码。

以Comparator接口为例，我们对前文“示例一”进行修改，利用Lambda表达式设置排序规则。

🟠 示例二：将整数列表中的元素倒序排列（使用Lambda表达式）。

在本示例中，我们定义一个整数列表，并将其中的元素倒序排列。

"TestBase.kt":

```kotlin
// 创建测试列表
val list = mutableListOf(1, 2, 3, 4, 5)


// 设置排序规则，将所有元素降序排列。
list.sortWith({ o1, o2 -> o2.compareTo(o1) })

// Lambda优化：如果Lambda表达式是方法的最后一个参数，可以将其移到括号外面。
list.sortWith { o1, o2 -> o2.compareTo(o1) }
```

此时Lambda表达式 `{ o1, o2 -> o2.compareTo(o1) }` 等价于前文“示例一”中的匿名内部类，语法更加简洁。

## 语法
Lambda表达式的标准格式为：

```text
{<参数1名称>: <参数1类型>, <参数2名称>: <参数2类型>, ... -> <函数体>}
```

在Kotlin语言中，无法直接用 `return` 关键字作为Lambda表达式的结果，我们约定将函数体末尾语句的值作为整个表达式的结果。若要在末尾语句之前返回结果，我们可以使用 `retrun@<标签>` 语法，例如：

```kotlin
list.sortWith { o1, o2 ->
    // 特殊规则，数值 `0` 排在最前面。
    if (o1 == 0) return@sortWith -1
    if (o2 == 0) return@sortWith 1

    o2.compareTo(o1)
}
```

前文“示例一”中的 `compare(o1: Int, o2: Int): Int` 方法转换为Lambda表达式后记作 `{ o1: Int, o2: Int -> o2.compareTo(o1) }` ，该表达式还可以进一步简化。

该表达式由函数式接口转换而来，编译器能够根据抽象方法的参数列表推断参数类型，因此我们可以省略参数类型，记作： `{ o1, o2 -> o2.compareTo(o1) }` 。

除了上述示例中的情况，Lambda表达式还有其他简化规则。

当Lambda表达式仅有单个参数时，其默认名称为 `it` ，我们可以省略参数声明，例如 `{ text -> System.out.println(text) }` 等价于 `{ System.out.println(it) }` 。

当Lambda表达式的函数体并未使用某个参数时，我们可以将其改为下划线( `_` )，例如： `{ text -> System.out.println("示例文本") }` 等价于 `{ _ -> System.out.println("示例文本") }` 。

若Lambda表达式仅调用单个方法，且该方法的参数列表、返回值与表达式完全相同，整个表达式可以简化为方法引用。

🟡 示例三：方法引用。

在本示例中，我们使用方法引用简化Lambda表达式。

第一步，将排序规则提取为一个方法。

"TestBase.kt":

```kotlin
fun reverse(a: Int, b: Int): Int {
    return b.compareTo(a)
}
```

第二步，创建测试数据并设置排序规则。

"TestBase.kt":

```kotlin
// 创建测试列表
val list = mutableListOf(1, 2, 3, 4, 5)

// 将Lambda参数传递给 `reverse()` 方法，并返回其结果。
list.sortWith { o1, o2 -> reverse(o1, o2) }

// 此时可以简化为方法引用形式： `::<方法名称>` 。
list.sortWith(::reverse)
```

在上述代码中，Lambda表达式 `{ o1, o2 -> reverse(o1, o2) }` 的参数列表、返回值与 `reverse()` 方法一致，因此我们可以将整个表达式替换为 `::reverse` 。

该示例中的 `reverse()` 是文件级方法，故方法引用的格式为 `::<方法名称>` ；如果需要引用实例方法，则语法为 `<对象名称>::<方法名称>` 或 `this::<方法名称>` 。

---

当我们在Kotlin中调用Java方法时，具有函数式接口的参数将被转换为对应的Lambda参数，以便我们使用lambda表达式，例如： `int sort(Comparator<? super E> c)` 方法对应 `sort { c: (Int, Int) -> int }` 方法。

如果Kotlin方法的参数为函数式接口，则不会进行自动转换，因为Koltin提供了 [🧭 高阶函数](#高阶函数) 特性，允许我们直接以函数作为参数。例如： `fun setListener(callback: Function<T, R>)` 不会生成对应的 `fun setListener { input: T -> R }` ，我们应当直接将该方法定义为 `fun setListener(onExec: (input: T) -> R)` 。

---

Lambda表达式不仅是函数式接口实例化的等价语法，在运行时它们的行为也存在差异。

如果Lambda表达式未引用任何外部变量，编译器只会创建单例对象，每次表达式被调用时均复用该对象，例如： `{ text -> println(text) }` 。函数式接口实例化每次都要创建对象，在此场景中占用内存更多。

如果Lambda表达式引用了外部变量，每次表达式被调用时都要创建对象保存变量引用，例如： `instance::method` ，此时Lambda表达式的性能与函数式接口实例化一致。


## 引用表达式
有时我们希望将Lambda表达式保存在变量中，以便多次复用。Kotlin语言支持将函数作为变量类型，我们可以直接声明变量保存对表达式的引用。

🟢 示例四：引用Lambda表达式。

在本示例中，我们使用Lambda表达式声明线程任务，并创建多个线程执行该任务。

"TestBase.kt":

```kotlin
// 将Lambda表达式的引用保存在变量中
val task: () -> Unit = { println("Hello, World!") }

// 创建线程，复用Lambda表达式。
Thread(task).start()
Thread(task).start()
```

## "this"关键字
当我们使用匿名内部类或Lambda表达式实现函数式接口时，都可以使用 `this` 关键字，但两种场景中该关键字的含义有所差异。

🔵 示例五： `this` 关键字。

在本示例中，我们辨析Lambda表达式与匿名内部类中 `this` 关键字的区别。

第一步，在测试类中编写一个成员方法，使用Lambda形式和匿名类形式分别创建Runnable接口的实例。

"TestBase.kt":

```kotlin
class Test() {

    fun testThis() {
        // 测试类实例的 `this` 引用
        println(this)


        // Lambda表达式中的 `this` 指向包含它的实例，即 `TestBase` 的实例。
        val lambda: () -> Unit = { println("this in lambda: $this") }
        lambda.invoke()


        // 匿名内部类中的 `this` 指向类的实例，可以访问其属性。
        val object1: Runnable = object : Runnable {

            // 匿名内部类可以拥有属性
            private val name = "Runnable"

            override fun run() {
                println("this in annonymous class: $this")
                // `this` 可以访问匿名内部类的属性
                println("get name by this: ${this.name}")
            }
        }
        object1.run()
    }
}
```

第二步，在 `main()` 方法中创建测试类的实例，并调用该成员方法。

此时运行示例程序，并查看控制台输出信息：

```text
net.bi4vmr.study.lambda.Test@7e9e5f8a
this in lambda: net.bi4vmr.study.lambda.Test@7e9e5f8a
this in annonymous class: net.bi4vmr.study.lambda.Test$testThis$object1$1@4554617c
get name by this: Runnable
```

根据上述输出内容可知：

匿名内部类的 `this` 指向自身实例，可以访问匿名内部类的成员变量；Lambda表达式的 `this` 等价于外部类的 `this` 。

虽然Lambda表达式在JVM内部仍有实例，但函数是无状态的，不能拥有成员变量，若将 `this` 指向Lambda实例没有实际意义；因此Lambda表达式中的 `this` 仍指向外部类，不具有独立的上下文，符合函数的定义。

## 捕获外部变量
Lambda表达式可能是作为异步回调接口使用的，并且可以访问外部变量，如果外部变量是方法的局部变量，需要在方法执行完毕后立即回收，为了使得Lambda表达式在方法回收后还能访问外部变量，编译器需要进行一些处理，这被称为变量捕获。

🟤 示例七：捕获外部变量。

在本示例中，我们尝试在Lambda表达式中访问和修改外部变量。

"TestBase.java":

```java
class Test2 {

    fun testCapture() {
        // 定义一个局部变量
        var num = 0

        // 定义Lambda表达式，尝试访问和修改外部变量。
        val lambda: () -> Unit = {
            // 访问外部变量
            println("Read `num` in lambda:[$num]")

            // 修改外部变量
            num++
        }

        // 执行Lambda表达式
        lambda()

        // 在Lambda表达式外部访问修改后的变量值
        println("Read `num` outside lambda:[$num]")
    }
}
```

此时运行示例程序，并查看控制台输出信息与界面外观：

```text
Read `num` in lambda:[0]
Read `num` outside lambda:[1]
```

Lambda会将外部变量的值复制一份保存在对象中，即使外部变量随方法结束而被销毁，Lambda表达式仍能访问变量的值。

与java不同，kotlin允许我们在lambda中修改外部变量的数值，如果它们是基本数据类型，kotlin编译器会自动将它们替换为IntRef等容器类型，修改本质上是修改容器中的数值，确保lambda内部的修改能够被外部观察到。
