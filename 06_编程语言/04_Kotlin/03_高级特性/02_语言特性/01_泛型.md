# 简介
普通变量的数据类型在声明阶段就确定了，例如 `val count: Int` 、 `val name: String` 等，调用者只能传入特定类型的数据。对于框架开发等场景，固定的数据类型不够灵活，框架需要提供较高的通用性，不应当限制调用者所使用的具体数据类型。

🔴 示例一：表示二维坐标。

在本示例中，我们定义一个类表示二维坐标，允许调用者传入整数、小数等多种形式的原始数据。

第一步，定义坐标类Location1，声明两个变量分别表示X轴和Y轴坐标，并提供一系列方法。

"Location1.kt":

```kotlin
class Location1 {

    var x: Any = 0
        private set

    var y: Any = 0
        private set

    // 设置坐标
    fun setXY(x: Any, y: Any) {
        this.x = x
        this.y = y
    }
}
```

由于普通变量只能声明为一种类型，若定义为Int就无法传入小数，若定义为Double则无法传入整数；在编写框架组件时，我们无法预知调用者将会传入何种元素，因此只能将坐标的数据类型定义为顶级父类Any。

第二步，切换到调用者视角，创建Location1实例并传入初始坐标，然后获取坐标值比较其是否与 `(100, 100)` 相等。

"TestGenerics.kt":

```kotlin
// 创建实例并设置初始坐标
val location = Location1()
location.setXY(100, 25.1082)

// 读取数据时需要判断具体的类型
when (val x = location.x) {
    is Int -> {
        // 将数据转换为具体类型再操作
        val resultX = x.compareTo(100) == 0
        println("x = 100? $resultX")
    }
    is Double -> {
        val resultX = x.compareTo(100.0) == 0
        println("x = 100.0? $resultX")
    }
    else -> {
        throw IllegalArgumentException("参数类型错误！");
    }
}

// 比较Y轴的数据时也需要判断类型，此处省略相关代码。
```

由于坐标的数据类型为Any，我们首先需要判断其具体类型，当数据类型非数字时抛出异常，仅当数据类型为数字时才能进行等值比较。

上述代码中的判断逻辑非常繁琐，降低了程序的可维护性；并且调用者可以传入任何类型的数据，直到程序运行阶段才会发现错误，效率较低。

```kotlin
// 随便传入一些非法数据也不会报错，直到读取数据时才能发现错误。
location.setXY("这是一些文本", BigDecimal("0.01"))
```

泛型(Generics)也被称为“参数化类型”，允许开发者将变量类型作为一个参数，无需指定具体的变量类型；当调用者使用这些变量时，再指明其具体类型。

泛型提高了程序的灵活程度，同时能够保证类型安全。虽然调用者可以自由设置泛型变量的数据类型，但这些类型只能在创建对象时被指定，后续调用者访问对象时若传入其他类型的数据，编译器能够及时地检测到错误并发出提示，不会将问题遗留到运行阶段。

定义泛型的语法如下文代码块所示：

```text
// 声明包含泛型的类
class 类名<泛型类型1，泛型类型2，...> {

    // 在变量或方法中使用泛型类型 ...
    val <变量名称>: <泛型类型1>
    val <变量名称>: <泛型类型2>
}
```

我们需要在类或接口名称后的尖括号( `<>` )中声明泛型参数，如果类或接口具有多个泛型参数，则以逗号( `,` )作为分隔符。每个泛型参数代表一种数据类型，我们可以将它们作为变量、方法形参或返回值的类型。

泛型参数的命名规则与常量相同，下文列表展示了常用的名称：

- `T` : Type，泛指任意类型。
- `E` : Element，表示集合中的元素。
- `K` : Key，表示键值对中的键名。
- `V` : Value，表示键值对中的键值。
- `N` : Number，表示数字。

如果我们创建的类继承自含有泛型参数的类，可以直接指明父类的泛型类型，也可以将泛型加入本类的泛型参数声明列表，由本类的调用者决定具体类型：

```text
// 父类泛型 `T1` 由子类决定
class 类名<T1，T2，...> : 父类名称<T1> {
    // 使用泛型参数...
}
```

当调用者创建泛型类的实例时，需要指明具体的类型：

```text
// 调用者需要用具体的类型替换占位符
val 变量名称: 类名<类型实参> = 构造方法<类型实参>();

// 示例：声明一个元素为Integer类型的List集合
val list: List<Integer> = MutableList<Integer>();

// 示例：简化写法，等号右侧的类型参数可根据左侧接收者推断得出，此时可省略右侧的类型声明。
val list: List<Integer> = MutableList<>();
```

在部分场景中，编译器可以根据上下文自动推导类型实参，此时我们可以省略创建实例时的类型实参声明，只保留尖括号即可。

Kotlin内置的List、Set、Map等集合就是包含泛型参数的类，它们能够容纳大量同一类型的元素，集合框架无需知晓具体的类型，因此使用泛型参数表示元素即可，具体类型由调用者在创建集合时决定。


# 基本应用
我们可以使用泛型参数表示坐标值的类型，解决前文示例中遇到的问题。

🟠 示例二：表示二维坐标（基于泛型）。

在本示例中，我们定义一个类表示二维坐标，允许调用者传入整数、小数等多种形式的原始数据。

第一步，新增Location2类，使用泛型描述X、Y轴坐标。

"Location2.kt":

```kotlin
class Location2<T1 : Any, T2 : Any>(
    x: T1,
    y: T2
) {

    // 变量的类型由调用者决定
    var x: T1 = x
        private set

    var y: T2 = y
        private set

    // 设置坐标
    fun setXY(x: T1, y: T2) {
        this.x = x
        this.y = y
    }
}
```

此时坐标变量 `x` 和 `y` 的类型是泛型，可由调用者决定具体的类型， `setXY()` 方法的参数与获取坐标方法的返回值也需要变更为对应的泛型类型。

第二步，切换到调用者视角，创建Location2实例并传入初始坐标，然后获取坐标值比较其是否与 `(100, 100)` 相等。

"TestGenerics.kt":

```kotlin
// 创建实例并设置初始坐标
val location = Location2(100, 25.1082)

// 读取数据时无需判断类型，与创建实例时的类型一致。
val x: Int = location.x
val resultX = x.compareTo(100) == 0
println("x = 100? $resultX")

// 比较Y轴的数据时无需判断类型，此处省略相关代码。
```

当我们构建Location2实例时，已经通过泛型指明了两个方向坐标变量的数据类型，因此 `getX()` 与 `getY()` 方法也会返回对应类型的数据，无需进行额外的类型判断。

如果我们调用Location2的 `setXY()` 方法时传入类型与泛型不匹配的数据，编译器就会提示数据类型不匹配，以便我们发现错误并进行修改。

```kotlin
// 参数类型与声明实例时的泛型类型不一致，编译阶段将会报错。
location.setXY("这是一些文本", BigDecimal("0.01"))
```


# 泛型方法
有时我们需要在方法级别定义泛型，将其作为方法参数或返回值，此类方法被称为“泛型方法”，通常是数据转换等工具方法。

前文“示例二”中的 `getX(): T1` 、 `setXY(x: T1, y: T2)` 等都不属于泛型方法，因为它们的泛型类型来自所属的类，并非由方法调用者指定。

🟡 示例三：泛型方法。

在本示例中，我们定义一个泛型方法，将数组转换为对应类型的列表。

第一步，定义泛型方法。

"TestGenerics.kt":

```kotlin
fun <F> arrayToList(array: Array<F>): MutableList<F> {
    val list = mutableListOf<F>()
    for (element in array) {
        list.add(element)
    }
    return list
}
```

`arrayToList()` 方法是一个泛型方法， `fun` 关键字与方法名称之间的 `<F>` 表示声明泛型参数 `F` ，`<F>` 的语法与类级别的泛型参数声明列表类似，若有多个泛型参数应以逗号( `,` )分隔。

第二步，调用泛型方法。

"TestGenerics.kt":

```kotlin
// 待转换的数组
val array = arrayOf(1, 2, 3, 4, 5)

// 调用泛型方法，类型由参数或返回值推断得出，此处为Integer。
val list: List<Int> = arrayToList(array)

// 我们也可以显式指定类型参数
// arrayToList<Int>(array)

println(list)
```

当我们调用泛型方法时，编译器将根据实参或返回值接收者推断泛型参数，我们也可以在方法名称与参数列表之间指明泛型类型。此处的测试数组 `array` 元素类型为Int，因此编译器推断泛型类型也为Int。


# 类型擦除
JVM中的泛型只在编译前有效，以便IDE和编译器进行语法检查提前发现错误；当源代码被编译为字节码后，泛型类型将被替换为Any类型或首个界限类型。

🟡 示例四：类型擦除。

在本示例中，我们通过反射获取泛型变量的类型，观察类型擦除的效果。

"TestGenerics.kt":

```kotlin
// 创建实例并设置初始坐标
val location = Location2(100, 25.1082)

// 尝试获取变量的类型
location::class.memberProperties
    .forEach {
        println("变量名称：[${it.name}] 变量类型：[${it.returnType}]")
    }
```

此时运行示例程序，并查看控制台输出信息：

```text
变量名称：[x] 变量类型：[T1]
变量名称：[y] 变量类型：[T2]
```

根据上述输出内容可知：

即使我们通过泛型指明了变量类型，程序运行时仍然无法获取到具体的类型。

类型擦除是JVM的历史遗留问题，使我们无法在运行时获取泛型参数的具体类型，为框架开发带来了挑战。我们可以通过后文内容所涉及的泛型界限与通配符改善类型擦除带来的问题，以便更好地使用泛型。
