# 简介
方法(Function/Method)是用来解决特定问题的代码组合，我们将若干语句定义为方法后，就可以多次重复调用，提高可读性与可维护性。

在Kotlin中，顶层方法可以直接定义在文件中，无需包裹在类里。定义方法使用 `fun` 关键字，语法如下文代码块所示：

```text
[访问修饰符] fun <方法名称>([参数列表...]): <返回值类型> {
    [方法体...]
}
```

🔷 访问修饰符

访问修饰符用于限定方法的可访问范围，可以是"public、internal、protected、private"或省略不写（默认为"public"），具体用法可参考相关章节。

🔷 返回值类型

返回值是方法的输出值，如果方法不需要向调用者汇报结果，此处应当填写"Unit"（等同于Java的"void"，可省略不写）；如果方法需要汇报结果，此处应当填写结果的数据类型，并在方法体中使用"return"语句指明结果。

🔷 方法名称

此处必须定义合法的标识符，我们通常使用驼峰命名法进行命名。

🔷 参数列表

参数是方法的输入值，传递给方法的参数可以有零个或多个，此时各参数间以逗号(",")隔开，每个参数由参数名称和参数类型组成（与Java顺序相反），例如： `(arg1: Int, arg2: String)` 。

<br />

调用方法的语法与Java相同：

```text
// 调用顶层方法
<方法名称>([参数...])

// 调用对象方法
<对象名称>.<方法名称>([参数...])
```

如果方法有返回值，我们也可以直接给变量赋值来接收返回值：

```text
val <用于接收返回值的变量> = <方法名称>([参数...])
```

# 方法的参数
通过方法的参数，我们可以把外部条件传递给方法，以便方法内部根据实际情况选择对应的操作，提高方法的普适性。

🔴 示例一：参数的基本应用。

在本示例中，我们定义一个方法，接收两个整型参数，功能是将它们的值累加求和，并将结果输出到控制台上。

"TestMethod.kt":

```kotlin
// 定义方法：计算两数之和。
fun plus(a: Int, b: Int) {
    val sum = a + b
    println("$a + $b = $sum")
}

fun main() {
    val x = 8
    val y = 6

    // 调用带参数无返回值方法，依次传入各参数。
    plus(x, y)
}
```

此时运行示例程序，并查看控制台输出信息：

```text
8 + 6 = 14
```

我们把定义方法时的参数称为"形参"，它们用于声明方法所需的参数数量与类型；我们在调用方法时传入的参数则被称为"实参"，这些参数是程序运行时真正被传递到方法中的值。

# 方法的返回值
在有返回值的方法中，我们需要使用"return"关键字标识输出到外部的结果，该结果可以是变量或固定值，且数据类型必须与返回值类型声明一致。

🟠 示例二：返回值的基本应用。

在本示例中，我们定义一个带返回值的方法，通过返回值汇报计算结果。

"TestMethod.kt":

```kotlin
// 计算两数之和
fun plus2(a: Int, b: Int): Int {
    val sum = a + b
    return sum
}

fun main() {
    val x = 8
    val y = 6

    // 调用带返回值方法，使用一个变量接收返回值。
    val summary = plus2(x, y)
    println("两数之和：$summary")
}
```

此时运行示例程序，并查看控制台输出信息：

```text
两数之和：14
```

方法的返回值不可省略，如果并不需要返回值，我们应当将返回值类型填写为"Unit"（或直接省略返回值类型声明）。

对于单表达式方法，Kotlin支持省略大括号和"return"关键字的简洁写法：

```kotlin
// 单表达式方法
fun plus3(a: Int, b: Int): Int = a + b
```

# 方法的重载
如果同一个作用域中包含两个或更多同名方法，但它们的参数数量或类型不同，这种情况被称为方法的"重载(Overload)"。当我们调用重载方法时，Kotlin会根据实参数量与位置来判断应当选择何种方法，形参与实参完全匹配的方法将被执行。

🟡 示例三：方法的重载。

在本示例中，我们定义两个同名方法，分别计算两个数和三个数的求和。

"TestMethod.kt":

```kotlin
// 计算两数之和
fun plus(a: Int, b: Int): Int = a + b

// 计算三数之和
fun plus(a: Int, b: Int, c: Int): Int = a + b + c

fun main() {
    val x = 8
    val y = 6
    val z = 10

    // 调用具有两个参数的"plus()"方法
    val sum1 = plus(x, y)
    println("两数之和：$sum1")
    // 调用具有三个参数的"plus()"方法
    val sum2 = plus(x, y, z)
    println("三数之和：$sum2")
}
```

此时运行示例程序，并查看控制台输出信息：

```text
两数之和：14
三数之和：24
```

# 默认参数
Kotlin支持为方法参数指定默认值，当调用者没有传入对应的实参时，方法将使用该默认值，这可以减少许多重载方法的定义。

默认参数的语法如下文代码块所示：

```text
fun <方法名称>(<参数名>: <类型> = <默认值>, ...): <返回值类型> {
    [方法体]
}
```

🟢 示例四：默认参数的基本应用。

在本示例中，我们定义一个方法，其中部分参数具有默认值，演示不同调用方式的效果。

"TestMethod.kt":

```kotlin
// 方法：打招呼，name有默认值。
fun greet(name: String = "World", times: Int = 1) {
    repeat(times) {
        println("Hello, $name!")
    }
}

fun main() {
    // 不传参数，全部使用默认值
    greet()
    // 只传第一个参数
    greet("Kotlin")
    // 传入全部参数
    greet("Alice", 2)
}
```

此时运行示例程序，并查看控制台输出信息：

```text
Hello, World!
Hello, Kotlin!
Hello, Alice!
Hello, Alice!
```

# 具名参数
Kotlin支持在调用方法时通过参数名称指定实参，被称为"具名参数(Named Arguments)"，这可以使代码更加清晰易读，并且允许以任意顺序传递参数。

🔵 示例五：具名参数的基本应用。

在本示例中，我们使用具名参数方式调用方法，演示参数顺序与可读性的提升。

"TestMethod.kt":

```kotlin
fun createUser(name: String, age: Int, email: String) {
    println("Name: $name, Age: $age, Email: $email")
}

fun main() {
    // 使用具名参数，顺序可以与定义不同
    createUser(
        age = 25,
        email = "alice@example.com",
        name = "Alice"
    )
}
```

此时运行示例程序，并查看控制台输出信息：

```text
Name: Alice, Age: 25, Email: alice@example.com
```

# 可变参数
Kotlin使用 `vararg` 关键字声明可变参数，允许调用者传入任意数量的实参。

可变参数的语法如下文代码块所示：

```text
fun <方法名称>(vararg <参数名称>: <参数类型>): <返回值类型> {
    [方法体]
}
```

可变参数具有以下限制条件：

- 可变参数只能出现在参数列表的最后一位，这也意味着每个方法最多只能拥有一个可变参数。
- 可变参数本质上是一个数组，在方法内部可以像数组一样使用。
- 调用者可以省略可变参数部分，此时方法内部接收到的数组长度为"0"，我们应当注意此类情况，防止出现数组越界异常。

🟣 示例六：可变参数的基本应用。

在本示例中，我们通过可变参数实现整数求和方法。

"TestMethod.kt":

```kotlin
// 计算若干整数之和。
fun calculateSum(vararg args: Int): Long {
    println("输入参数为：${args.toList()}")
    var sum = 0L
    for (arg in args) {
        sum += arg
    }
    return sum
}

fun main() {
    // 传递可变参数（通常用法）
    val result1 = calculateSum(1, 2, 3)
    println("Summary is $result1.")

    // 使用展开运算符将数组展开为可变参数
    val arr = intArrayOf(1, 2, 3)
    val result2 = calculateSum(*arr)
    println("Summary is $result2.")

    // 传递可变参数（0个参数）
    val result3 = calculateSum()
    println("Summary is $result3.")
}
```

此时运行示例程序，并查看控制台输出信息：

```text
输入参数为：[1, 2, 3]
Summary is 6.
输入参数为：[1, 2, 3]
Summary is 6.
输入参数为：[]
Summary is 0.
```

# 回调方法
回调方法(Callback)是一种常用的设计模式，能够实现异步编程。在Kotlin中，方法是"一等公民"，可以直接将Lambda表达式或函数引用作为参数传递，无需像Java那样定义接口。

函数类型的语法为 `(参数类型...) -> 返回值类型` 。

🟤 示例七：回调方法的基本应用。

在本示例中，我们定义一个方法，接收Lambda表达式作为回调，并在异步操作完成后触发。

"TestMethod.kt":

```kotlin
// 保存文件（模拟异步操作）
fun saveFile(callback: (Boolean, String) -> Unit) {
    Thread {
        Thread.sleep(2000L)
        // 异步操作完成，触发回调。
        callback(true, "OK.")
    }.start()
}

fun main() {
    println("Test method start.")
    // 调用方法，传入Lambda表达式作为回调。
    saveFile { result, message ->
        println("Callback triggered. Result: $result, Message: $message")
    }
    println("Test method end.")
    // 等待异步线程结束
    Thread.sleep(3000L)
}
```

此时运行示例程序，并查看控制台输出信息：

```text
Test method start.
Test method end.
Callback triggered. Result: true, Message: OK.
```

# 内联方法
## 简介
在Kotlin中，将Lambda作为参数传递时，会在运行时产生额外的对象和调用开销。"内联方法(Inline Function)"通过 `inline` 关键字声明，编译器会将内联方法的代码直接"内联"到调用点，从而消除Lambda带来的性能损耗。

内联方法的语法如下文代码块所示：

```text
inline fun <方法名称>(<参数名>: <Lambda类型>): <返回值类型> {
    [方法体]
}
```

🔴 示例八：内联方法的基本应用。

在本示例中，我们定义一个内联方法，并与普通方法进行对比。

"TestMethod.kt":

```kotlin
// 内联方法：测量代码块的执行时间
inline fun measureTime(block: () -> Unit): Long {
    val start = System.currentTimeMillis()
    block()
    return System.currentTimeMillis() - start
}

fun main() {
    val elapsed = measureTime {
        // 模拟耗时操作
        var sum = 0L
        for (i in 1..1_000_000) {
            sum += i
        }
        println("计算完毕")
    }
    println("耗时：${elapsed}ms")
}
```

此时运行示例程序，并查看控制台输出信息：

```text
计算完毕
耗时：XXms
```

> 🚩 提示
>
> 内联方法适合参数中含有Lambda的工具性方法（如测量时间、资源管理等）。如果方法体本身较大，内联可能导致生成的字节码膨胀，此时不建议使用 `inline` 修饰。
